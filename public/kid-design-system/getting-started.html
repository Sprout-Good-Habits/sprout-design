Getting Started
Everything you need to start building with the Sprout design system — token setup, page templates, and the core rules that keep things consistent.

1. Link the Token Stylesheet

Every page must link to tokens.css — never redeclare token values in your own CSS.

<!-- For pages inside sprout-hub --> <link rel="stylesheet" href="/brand/tokens.css"> <!-- For standalone files outside the hub --> <link rel="stylesheet" href="https://sprout-hub-preview.vercel.app/brand/tokens.css">

2. Page Template

Use this skeleton for new documentation or component pages in the hub.

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Sprout Hub — Section — Page</title> <link rel="stylesheet" href="/brand/tokens.css"> <link rel="stylesheet" href="/assets/shared.css"> <link rel="stylesheet" href="/assets/components.css"> <style>/* page-specific CSS */</style> </head> <body> <!-- sidebar, main, page-content --> <!-- doc-section blocks --> <script src="/assets/shell.js"></script> </body> </html>

3. Core Rules

These six rules are non-negotiable. Every implementation must follow them.

1

Never hardcode colors

Use var(--token-name), not #hex. If you catch yourself writing a hex value, find the matching token.

2

Never redeclare tokens

Link to tokens.css via <link> — do NOT copy token definitions into :root or <style>.

3

Never inline shared assets

If an SVG exists in /brand/assets/, reference it with <img src="..."> — never paste its markup.

4

Never invent components

Check the component library first. If it exists, use the documented pattern exactly.

5

Always pair font-size with line-height

--font-size-text-md goes with --line-height-text-md. Never use one without the other.

6

Always use the spacing scale

No magic numbers like padding: 15px. Use var(--spacing-xl) (16px) or var(--spacing-lg) (12px).
