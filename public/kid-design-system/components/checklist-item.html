Checklist Item
Represents a single step in a parent-assigned checklist quest (e.g. morning routine). Each item shows a task, what proof is required, and the current completion state. Appears as a card in a Tier 2 carousel or a compact row in a Tier 3 list.
  • ✓ Use inside a checklist quest lobby to show each step the kid needs to complete.
  • ✓ Use when a parent has configured required proof (photo, video, or chat) for each step.
  • ✗ Don't use for generic tasks on the home screen. Use Task Card instead.
  • ✗ Don't use for non-quest items or items without a proof requirement.

Anatomy

Card (Tier 2)

  • Emoji avatar (required): uses the shared CardAvatar component sized down to 88px (40px emoji) via a scoped override on .cl-card .card-avatar.
  • Task name (required): text-lg (18px) semibold. The name of the checklist step assigned by the parent.
  • Action button (required): 42px tall, full width, 12px border-radius, with a 20px proof icon and an uppercase label. Label is always "TAKE PHOTO" regardless of proof type. Changes to "TRY AGAIN" in warning state. In done state, the button fills solid fg-disabled (#a4a7ae) with text-disabled (#717680) text and no border/shadow.
  • Status flag (conditional): a 40px bookmark ribbon anchored top-right, shown only in warning (yellow with ⚠️) and done (green with ✅️) states. Uses the same StatusFlag pattern as Task Card.

Row (Tier 3)

  • Task emoji (required): 24px TossFace emoji on the left.
  • Task name (required): text-md (16px) regular, flex-fills the row. Strikethrough when done.
  • Proof icon (to-do only): 24px TossFace emoji on the right showing the required proof type (📷️ camera, 📹️ video, 💬 chat).
  • Status emoji (done/warning): 24px TossFace emoji replacing the proof icon. ✅️ for done, ⚠️ for warning.
  • Expanded action area (warning only): the row wraps in a bg-secondary container and reveals a retry message + full-width btn-secondary btn-md "TRY AGAIN" button below.

Placement

Checklist items live inside the quest lobby screen, below the Sprout character and progress bar.

  • ✓ Tier 2 (ages 7-9): Display cards in a horizontal carousel with 12px gaps. Cards snap to center. The next card peeks from the right edge to suggest scrollability.
  • ✓ Tier 3 (ages 10-13): Stack rows vertically with 16px gaps inside the lobby container. The list scrolls vertically within the content area.
  • ✗ Don't mix card and row formats in the same lobby. Tier determines the format.
  • ✗ Don't show more than one expanded warning row at a time in the Tier 3 list.

State Transitions

The checklist item lifecycle follows the proof submission and parent review flow:

  1. To do: Kid has not yet submitted proof. Tapping the action button opens the camera, video recorder, or chat depending on proof type.
  2. Pending review: Proof has been submitted but not yet reviewed by the parent. (Visually identical to "to do" for the kid; the progress bar tracks this.)
  3. Done: Parent has approved the proof. The item is marked complete.
  4. Warning: Parent has rejected the proof. The item shows a retry state with a "TRY AGAIN" action.

Content

  • Task names are parent-authored and should be short (under 30 characters). The component truncates with ellipsis if needed.
  • Button labels are fixed system copy: "TAKE PHOTO" (all proof types) or "TRY AGAIN" (warning state). The proof icon differentiates the proof type, not the label.
  • Retry messages follow the pattern: "Take a new [proof type] to complete this" or "Let's talk again to complete this".
  • The task emoji is set by the parent or auto-assigned from a predefined set matching common routine tasks.

Accessibility

  • Role: Each item uses role="article" with an aria-label describing the task name and current state (e.g. "Get dressed, to do, take photo").
  • Keyboard: Tab moves focus between checklist items. Enter activates the proof action or retry button.
  • Done state: The action button has aria-disabled="true" to indicate it cannot be activated.
  • Warning state: Use aria-live="polite" on the retry message so screen readers announce when an item transitions to warning.
  • Touch targets: The Tier 2 card button and Tier 3 expanded retry button both exceed the 44px minimum. The entire Tier 3 row is tappable.
  • Status flag: Flag icons are decorative (aria-hidden="true"). State is conveyed via the aria-label on the parent element.
