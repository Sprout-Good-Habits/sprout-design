Message Composer
A multi-modal input bar for composing text messages, voice recordings, and media attachments in the chat experience with Sprout.

When to use

  • ✓ Use for the primary chat input in conversations with Sprout
  • ✓ Use when the child needs to send text, voice, or image content
  • ✓ Use when media attachments need to be previewed before sending
  • ✕ Do not use for simple form text entry (use Input instead)
  • ✕ Do not use for search fields
  • ✕ Do not use outside of a chat/conversation context

Anatomy

The Message Composer is composed of an outer layout row and an inner bubble with contextual content.

  8. Play/Pause button — 32px gray circle. Previews recorded audio (send-audio state).
  9. Waveform — 2px bars, 2px gap. Animated during listening, static in send-audio.
  10. 7Media row — Horizontal scroll of 120px image cards with remove buttons and loading overlays.

Accessibility

  • Text input: The textarea must have aria-label="Message" or equivalent. Placeholder text alone is not sufficient for screen readers.
  • Buttons: All icon-only buttons (send, stop, play, mic, attach, remove) must have descriptive aria-label attributes.
  • Keyboard: Tab moves focus between the textarea and action buttons. Enter submits the message (or Shift+Enter for newline).
  • Voice states: Use aria-live="polite" to announce state transitions (listening, transcribing, transcription complete).
  • Media attachments: Each media card should announce its type and provide an accessible remove action. Loading state should be conveyed via aria-busy.
  • Touch targets: All action buttons meet or exceed the 44x44px minimum. The attach button is 48px, action circles are 32-36px with adequate padding.
  • Contrast: Placeholder text (--text-placeholder-subtle) is decorative and not relied upon for information. Input value text (--text-secondary) on --bg-primary passes WCAG AA.
