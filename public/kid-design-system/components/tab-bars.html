Tab Bars
The tab bar is the primary bottom navigation for the kid-facing app. It provides persistent access to the five top-level sections (Home, Calendar, Projects, Wishlist, and Profile) using colorful TossFace emoji icons.
  • ✓ Use as the persistent bottom navigation in the kid app.
  • ✓ Always show all five tabs. Never hide or conditionally remove tabs.
  • ✓ Pair with the Toolbar at the top for complete app chrome.
  • ✗ Don’t use for secondary or nested navigation. Use Segmented Control instead.
  • ✗ Don’t use in the parent app. The tab bar is kid-facing only.

When to use

The tab bar is the only sanctioned primary navigation pattern in the kid-facing app. It stays visible across all top-level screens so kids always have a clear path home.

  • Always visible. The tab bar persists across all five main sections. It hides only during full-screen quest/exercise flows.
  • Five fixed tabs. Home, Calendar, Projects, Wishlist, Profile. The order and set are fixed and should not change.
  • Active state. The current tab gets a lighter pill background (bg-grass-tertiary) with a subtle shadow. Inactive tabs have no background or shadow.

Anatomy

Each tab consists of an icon slot and a label. The bar itself sits at the bottom of the viewport with a rounded top edge and extra bottom padding for the home indicator.

Placement

The tab bar is always anchored to the bottom of the screen. The 32 px bottom padding accounts for the iOS home indicator safe area.

  • Position. Fixed to the bottom edge of the viewport. Content scrolls behind it.
  • Pairing. Always appears with a Toolbar at the top. Together they form the persistent app chrome.
  • Hiding. The tab bar hides during immersive flows like quests and exercises where the child should be focused on the task.

Content guidelines

Tab labels and icons should be immediately recognizable to children.

  • Labels. Single word, title case. Maximum 8 characters to prevent truncation on small screens.
  • Icons. TossFace emoji only (never system emoji). Each icon should be visually distinct and semantically connected to the section.
  • Order. Home → Calendar → Projects → Wishlist → Profile. This order is fixed.

Accessibility

The tab bar must be fully navigable by keyboard and assistive technology.

  • Wrap the tab bar in a <nav> element with aria-label="Main navigation".
  • Each tab uses role="tab". The active tab gets aria-current="page".
  • Support Tab to enter the bar, then ArrowLeft / ArrowRight to move between tabs.
  • Each tab item meets the 44×44px minimum touch target.
  • Icon images use descriptive alt text (e.g., "Home icon").
