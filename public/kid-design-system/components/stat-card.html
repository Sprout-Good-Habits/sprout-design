Stat Card
A compact, tier-labelled metric tile used on session result screens to celebrate accuracy and speed.
  • ✓ Use a stat card to summarise one performance metric (accuracy or time) at the end of a quest or session.
  • ✗ Don’t use a stat card for interactive content, counters inside a form, or as a button.

Variants

There are two variants, Target and Duration, each reporting a different metric. A typical result screen pairs one Target card (yellow) with one Duration card (green) side by side.

A note on colour. Yellow (Target) and green (Duration) are used decoratively here, to visually separate the two tiles on the same result screen. They do not carry the semantic weight of “warning” or “success”, a yellow Target at 20% is not a warning state and a green Duration at 10:00 is not a success state. That is why the card uses raw palette tokens (--yellow-400, --green-400) and not semantic tokens like --bg-warning-primary or --bg-success-primary. The mapping between variant and colour is fixed: Target is always yellow and Duration is always green, don’t override it per instance.

Target

Uses the 🎯 bullseye icon. The value is always an accuracy percentage with no decimals. The tier label reframes the raw number as encouragement, so even low scores feel recognised rather than punished.

  • ✓ Use Target for the share of correct answers, formatted as %.
  • ✗ Don’t use Target for counts, time values, XP, or gems, it is strictly an accuracy percentage.

Tiers

Each metric type has four tiers. The label and thresholds change, but the card structure stays constant. This keeps the card feeling celebratory at every tier, rather than shifting between “good” and “bad” colours.

Tier thresholds are based on percentage correct. Below 20% the session is treated as incomplete and does not render a stat card.

Anatomy

A stat card is always two stacked blocks inside a coloured outer frame. The outer colour is driven by the variant (.stat-card--target is yellow, .stat-card--duration is green), the inner block is always white.

GREAT
🎯 80%
  1. Outer frame, .stat-card, 2px of padding in the colour modifier, acts as the border.
  2. Label strip, .stat-card-label, uppercase 12px bold white text naming the tier.
  3. Value body, .stat-card-value, white background, 12px padding, contains the icon and value.
  4. Icon, .stat-card-icon, 24px TossFace emoji (🎯 for Target, ⏲️ for Duration).
  5. Value, .stat-card-number, 16px regular secondary text, percentage (Target) or time (Duration).

Placement

Stat cards live on result screens, placed in a row directly above the primary action. Pair one Target card with one Duration card for the standard summary, and always lay them side by side, never stacked. Use .stat-card-row as the wrapper, the two cards flex to fill the row equally and each ends up at roughly 181px on a typical 370px content width.

  • ✓ Pair two cards side by side using .stat-card-row, so both metrics read at a glance.
  • ✗ Don’t stack cards vertically, they lose their visual rhythm and push the primary action below the fold.
  • ✗ Don’t override the colour on a per-instance basis, Target is always yellow and Duration is always green.

Content

Stat card copy should:

  • Be one or two words for the tier label, always in uppercase
  • Stay positive at every tier, never use “Bad”, “Poor”, or “Slow”
  • Use % for Target values, with no decimals
  • Use M:SS or MM:SS for Duration values, never display seconds alone
  • Keep labels short enough to fit without truncation when the card is rendered at its maximum 181px width

Accessibility

Stat cards are summary content, not interactive controls, but they still need to read clearly for screen reader users and meet colour contrast guidance.

  • Semantics: Wrap each card in a role="group" with an aria-label combining the tier and value, for example aria-label="Accuracy, great, 80 percent". The visible emoji icon should be marked aria-hidden="true" so it is not announced twice.
  • Keyboard: Stat cards are non-interactive, so they are not in the tab order. The primary action button below the cards remains the only focusable control on the result screen.
  • Contrast: The white text on --yellow-400 and --green-400 is below AA for normal text, so the label strip uses 12px bold (large-text threshold) and is paired with a white interior value for the actual data. Never place the percentage or time value on the coloured strip.
  • Colour independence: The tier is always spelled out in the label strip, so users who cannot distinguish yellow from green still get the full information.
  • Motion: When cards animate in on the result screen (scale or slide), wrap the animation in a prefers-reduced-motion media query and fall back to an instant fade.
  • Counter animation: On result screens, the numeric value inside each card counts up from 0 to its final value on entry. The shared helper lives at /components/stat-card.js and exposes SproutStatCard.animateRow(rowEl). See the Result Screen pattern for timing, format rules, and a full usage example.
