Alert
A centered modal card that interrupts the user with a blocking decision: confirming a destructive action, acknowledging an outcome, or choosing between two paths forward.
  • ✓ Use an alert when the user must make a blocking decision before continuing.
  • ✓ Use to confirm destructive actions like deleting, leaving, or logging out.
  • ✗ Don’t use for post-action feedback tied to a single answer. Use Feedback Banner instead.
  • ✗ Don’t use for passive notifications that don’t need a response. Use Toast instead.
  • ✗ Don’t use for multi-step content or form inputs. Use Sheet instead.

Types

The alert is a neutral card. Its visual treatment is always the same. Choose a layout based on how many buttons you need and how long the labels are. Sentiment is carried by the buttons inside, not the alert container.

Stacked Default

Use when you have one, two, or three actions, or when any button label is long enough to wrap. Full-width buttons stack vertically with the primary action on top.

Side-by-side Default

Use only when you have exactly two short buttons, typically a cancel paired with a confirm. The secondary action sits on the left, the primary on the right. If either label is long, use stacked instead.

Anatomy

  • Icon (optional): 40×40px TossFace emoji above the content. Use to reinforce tone (warning, celebration, deletion).
  • Label (required): text-md (16px) bold, centered. States the decision the user needs to make.
  • Description (optional): text-md (16px) regular, centered. Explains the consequence or adds context.
  • Primary action (required): the main path forward. In stacked layout it sits on top; in side-by-side it sits on the right.
  • Secondary action (optional): cancel, dismiss, or alternative path. Never place the only action as a secondary button.
  • Destructive action (optional): use btn-destructive for actions that can’t be undone.

The container is a fixed 370px white card with a 2px border, 12px radius, and a large drop shadow that lifts it off the backdrop.

Content Composition

All elements except the label and one action button are optional. Compose only what the moment needs.

Placement

The alert is always presented as a modal, centered over a dimmed backdrop that covers the entire viewport. The content behind it is not interactive until the user picks an action.

  • ✓ Center the alert vertically and horizontally on screen.
  • ✓ Cover the viewport with a dimmed scrim at roughly 40% opacity.
  • ✓ Animate the alert with a short fade + scale-in. Respect prefers-reduced-motion.
  • ✗ Don’t place an alert inline within scrolling content.
  • ✗ Don’t stack alerts. Resolve one before showing another.

Content

Alert copy should:

  • Ask the decision clearly in the label: “Leave this lesson?”, “Delete draft?”
  • State the consequence in the description: “Your progress will be saved”, “This can’t be undone”
  • Use verb-first button labels that match the decision (“Leave”, “Delete”, “Continue”), not “Yes” / “No”
  • Keep the label under 40 characters and descriptions under 80 characters
  • Put the recommended or safe action as primary; destructive options use btn-destructive
  • Avoid blame or jargon. Kids should feel in control, not scolded

Accessibility

  • Role: Use role="alertdialog" with aria-modal="true" so assistive tech treats the alert as a blocking dialog.
  • Labelling: Point aria-labelledby at the label element and aria-describedby at the description element.
  • Focus management: When the alert opens, move focus to the primary action (or the least-destructive action). Trap focus inside the alert while it’s open; restore focus to the trigger when it closes.
  • Keyboard: Tab cycles through buttons, Enter or Space activates the focused button, and Escape triggers the cancel/dismiss action when present.
  • Touch targets: All buttons meet the 44×44px minimum touch target.
  • Contrast: Label and description pass WCAG AA on the white container; button contrast is handled by the Buttons component.
  • Motion: If animating the open/close transition, respect prefers-reduced-motion by disabling the fade + scale.
  • Backdrop: The scrim should not be the only way to dismiss. Always provide a visible cancel action for destructive alerts.
