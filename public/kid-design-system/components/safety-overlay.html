Safety Overlay
A full-screen dark overlay with backdrop blur that appears when the on-device SFW (safe-for-work) check fails during a camera recording session. It obscures the camera feed and shows a non-alarming “Camera Paused” message to the child.
  • ✓ Use when the on-device SFW check detects inappropriate content in the camera feed.
  • ✓ Keep the messaging calm and non-alarming — the child should not feel scared or in trouble.
  • ✗ Don’t add interactive elements (buttons, links) — the overlay clears automatically when the camera feed is appropriate again.
  • ✗ Don’t use for general error states — use Feedback Banner instead.

Hierarchy

The safety overlay has a single visual hierarchy. It always uses a dark, blurred backdrop to fully obscure the camera feed regardless of content.

Default

The only variant. A dark semi-transparent background (85% opacity) with a 20px backdrop blur ensures the camera feed is completely obscured. Centered content uses white text on dark to maintain readability and a calm tone.

Anatomy

  • Icon (required) — 56px emoji. Provides an immediate visual signal that the camera is paused. Use a neutral, non-scary icon.
  • Heading (required) — Text-lg (18px) bold, white. Short, calm label like “Camera Paused”.
  • Description (required) — Text-sm (14px) regular, white at 70% opacity. Explains what happened and what to do next. Max width 260px to keep lines readable.
  • Fine print (optional) — Text-xs (12px) regular, white at 40% opacity. Technical note for transparency, e.g. “SFW check — frame not sent to server”.

Placement

The safety overlay fills its parent container using position: absolute; inset: 0. The parent (camera preview) must have position: relative and overflow: hidden.

  • ✓ Always position the overlay as a direct child of the camera preview container.
  • ✓ Use z-index: 40 to ensure it sits above camera controls and recording indicators.
  • ✗ Don’t use position: fixed — the overlay should be scoped to the camera area, not the full viewport.

Content

Safety overlay copy should:

  • Be calm, reassuring, and never blame the child — use neutral phrases like “Camera Paused” rather than “Inappropriate content detected”.
  • Give a clear, simple instruction for what to do next — “Point the camera at the task and we’ll continue.”
  • Avoid technical jargon in the heading and description — save technical details for the optional fine-print line.
  • Keep the description to one or two short sentences maximum.
  • Use the fine-print line to communicate privacy-relevant info (e.g. that the frame is not sent to a server) if needed for transparency.

Accessibility

  • Live region: The overlay should use role="alert" and aria-live="assertive" so screen readers announce the pause immediately when it appears.
  • Label: Add aria-label="Camera paused" to the overlay container for additional context.
  • No interaction required: The overlay has no interactive elements — it clears automatically when the camera feed is appropriate again, so no keyboard handling is needed.
  • Contrast: White text on the dark overlay background (85% black) exceeds WCAG AAA contrast ratios. The 70% opacity description text also passes AA.
  • Motion: If using a fade-in animation, respect prefers-reduced-motion by disabling the transition.
