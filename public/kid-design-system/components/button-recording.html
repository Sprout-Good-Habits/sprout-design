Button Recording
Circular 72×72 start/stop recording control with Sprout’s signature raised shadow and press animation. Two types (Primary, Secondary) and two actions (Start, Stop) cover every recording surface in the product. Secondary is the default for camera and video viewfinder overlays (Camera, Video) because its white fill with a dark indicator reads clearly without dominating the feed. Reach for Primary only when the button needs heavier brand emphasis.

  • ✗ Don’t use a recording button as a general-purpose action. Use Buttons or Button Utility instead.

Types

Two types of recording button give you coverage for both neutral and busy surfaces. Both types support the same Start and Stop actions, states, and sizing. Only the Start fill treatment differs — the Stop state is visually identical across both types (see the Token Reference note below).

Primary Neutral

A solid, confident fill variant. Reach for Primary when the button needs heavier brand emphasis or sits on a plain neutral surface where a white button would disappear. It is not the default.

  • ✓ Use Primary on plain surfaces where you want the record button to feel like the most important control on the screen.
  • ✗ Don’t use Primary on busy imagery, camera viewfinders, or branded backgrounds. Use Secondary instead.

Secondary Neutral Default

The default recording control for camera and video viewfinder overlays. White fill, 2px gray stroke, --fg-secondary indicator for the Start state so the button recedes into busy backgrounds. The .btn-record-icon slot is consumer-pluggable: drop in a filled circle for the canonical Figma V Start dot, a camera SVG for photo capture, or any other inline SVG — it paints via currentColor. The Stop state is visually identical to Primary Stop, since stopping a recording should always read as the same decisive action.

  • ✓ Use Secondary over photo, video, or branded backgrounds. This is the right choice for the Camera and Video patterns.
  • ✓ Override the .btn-record-icon slot with whatever inline SVG fits the context — dot, camera, mic, custom glyph. The component paints it via currentColor.
  • ✗ Don’t pair Secondary with Primary on the same surface. Pick one type per screen.

Actions

Recording buttons always represent one of two mutually exclusive actions. Toggle between them as the recording state changes, and use aria-pressed to communicate the toggle to assistive tech.

Start recording Positive

Shown before the user has begun recording. Uses the brand blue fill on Primary, and a neutral --fg-secondary indicator dot on Secondary so the button recedes into busy or branded backgrounds.

Stop recording Destructive

Shown while a recording is active and can be ended. Uses red to signal the end of a time-sensitive action. Kids immediately understand “red square” as stop.

  • ✓ Always flip from Start to Stop (and back) on the same button position so the press target stays put.
  • ✗ Don’t show Start and Stop side by side. The user should only ever see one.

Interaction

The button lifts 4px off the surface by default and compresses flush on press, giving tactile feedback that works especially well for kids. Transitions use a 0.1s cubic-bezier(0.25, 0.1, 0.25, 1) curve, matching Buttons exactly.

On press, the box-shadow collapses to none and the button translates translateY(4px) so the bottom edge lands in the same place.

Content

Recording buttons don’t carry visible text: the icon alone must communicate the action. The accessible name comes from aria-label, which should:

  • Start with a verb, like “Start recording” or “Stop recording”
  • Describe the action that will happen when pressed, not the current state
  • Be short (2–3 words)
  • Be in sentence case

Accessibility

Recording buttons are high-stakes controls that a kid might press when they can’t read yet, so accessibility matters even more than for regular buttons. All recording buttons must meet WCAG AA standards.

  • Keyboard: All recording buttons are focusable and activated with Enter or Space. Use native <button> elements, never <div> with click handlers.
  • aria-label: Every recording button must have an aria-label describing the action (“Start recording” or “Stop recording”) because the button has no visible text.
  • aria-pressed: When the same button toggles between Start and Stop, set aria-pressed="true" while a recording is active and aria-pressed="false" (or omit) otherwise.
  • aria-busy: Add aria-busy="true" during loading states so screen readers announce that the action is in progress.
  • Focus ring: A visible 3px focus ring shows on keyboard navigation. Do not remove :focus-visible outlines. The ring color shifts to red on Stop variants for semantic clarity.
  • Contrast: Primary variants (white icon on --brand-500 or --red-500) pass AA for non-text contrast. Secondary Start (--fg-secondary dot on white) passes AA for non-text contrast at the 14px indicator size.
  • Touch targets: The 72×72px size comfortably exceeds the 44×44px minimum touch target guideline, giving kids room to press confidently.
  • Motion: The press animation (translateY) and the loading spinner respect prefers-reduced-motion and stop animating when the user has requested reduced motion.
