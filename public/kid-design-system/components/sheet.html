Sheet
A sheet slides up from the bottom of the screen to present focused content or actions without navigating away from the current view.
  • ✓ For focused choices that don’t need a full screen (e.g. selecting a duration, picking an app).
  • ✓ For confirmation actions before spending currency or making changes.
  • ✓ When the user should remain aware of the screen behind them.
  • ✗ Don’t use for complex multi-step flows — navigate to a full screen instead.
  • ✗ Don’t use for passive information that doesn’t require action — use a toast or banner.

Title content

Title copy should:

  • Be just a few words
  • Put the most important words first
  • Summarise or introduce the content it relates to (imagine the user only reads the title)
  • Be a statement rather than a question (unless the content is asking the user something)
  • Be in sentence case (only capitalise the first letter of the first word)
  • Have no full stop

Anatomy

A sheet has three structural zones. The top and footer stay pinned while the body scrolls when content overflows.

Structure

Title

Optional description
Content area (scrollable)
  • Handle — 36×4px grabber bar. Visual affordance for swipe-to-dismiss.
  • Title (required) — display-md (36px/44px bold, -0.02em tracking). Always an <h1>.
  • Description (optional) — text-sm (14px/20px regular), --text-quaternary.
  • Close button (required) — 24px icon inside a 44px tap target. Always present.
  • Body — Scrollable content area. Can contain list items, forms, text, or any component.
  • Footer — Sticky action bar. Shows a border-top divider when the body is scrollable. Hidden or borderless when content fits.

Scrolling the content

The height of the bottom sheet automatically adjusts to the content inside it. But if the content is longer than the maximum height that the bottom sheet can expand to, the content will overflow and become scrollable.

Footer variations

The footer supports several patterns depending on the action type.

Primary
Primary + Secondary
Destructive
Destructive + Secondary
  • Primary — the default. Single full-width primary button.
  • Primary + Secondary — primary on top, secondary below. Use when there’s a clear alternative action (e.g. “Confirm” / “Cancel”).
  • Destructive — single full-width destructive button. Use for irreversible actions like deleting data.
  • Destructive + Secondary — destructive on top, secondary below. Gives the user a safe way out before committing to a destructive action.

Empty state

When a sheet has no content to display, show a Toss Face icon with a short message. The CTA should be disabled or hidden.

Stacking

Sheets can stack. A sheet may open another sheet on top of itself. The previous sheet stays visible underneath with an additional scrim layer.

  • ✓ Limit stacking to two levels deep — more than that is confusing.
  • ✓ Dismissing the top sheet reveals the one below. Dismissing both returns to the original screen.
  • ✗ Don’t stack sheets for sequential steps — navigate to a new screen instead.

Dismissing a bottom sheet

Users can dismiss the bottom sheet by:

  • Tapping or clicking the close button
  • Taking an action within the bottom sheet content
  • Dragging or swiping the bottom sheet down

Animation

Sheets animate in and out to maintain spatial awareness.

  • Entry — slide up from the bottom over 0.5s cubic-bezier(0.5, 1.8, 0.3, 0.8) (spring). The scrim fades in simultaneously.
  • Exit — slide down over 0.35s ease-in. Slightly faster for a snappier feel.
  • Snap points — none. The sheet is either open or closed. No intermediate half-height positions.

Accessibility

  • Role: Use role="dialog" with aria-modal="true" on the sheet container.
  • Label: Set aria-labelledby pointing to the title’s id.
  • Focus trap: Keyboard focus must stay within the sheet while open. Tab should cycle through interactive elements.
  • Escape: Pressing Esc dismisses the sheet and returns focus to the trigger element.
  • Close button: Must have aria-label="Close".
  • Scrim: Assistive tech should not be able to interact with content behind the scrim.
  • Touch target: Close button has a 44×44px tap target, meeting accessibility guidelines.
