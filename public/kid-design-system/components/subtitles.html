Subtitles
A text bubble that delivers short conversational or feedback messages. Subtitles can be used standalone or composed inside Conversational Feedback alongside an Avatar Portrait or Avatar.
  • ✓ Use for short encouragement, feedback, or conversational messages.
  • ✓ Use Centered for simple messages, Left aligned for longer text, Left aligned + icon when an emoji adds context.
  • ✓ Use the Overlay context when placed on top of rich backgrounds or images.
  • ✗ Don’t use for long-form content — text is capped at 120px height because messages rotate like subtitles.
  • ✗ Don’t add buttons or interactive elements inside subtitles.

When to use

Subtitles deliver brief, contextual messages to the user.

  • Standalone — As a lightweight feedback message on its own, e.g., a streak celebration or coaching nudge.
  • Inside Conversational Feedback — Paired with an Avatar Portrait or Avatar to create a full conversational moment with character presence.
  • Overlay context — On top of dark or busy backgrounds where the default style wouldn’t have enough contrast.

When not to use

  • Don’t use for full chat messages — use Message Bubble instead.
  • Don’t use for system alerts or errors — use Toast instead.
  • Don’t use for long text that exceeds the 120px max height — the content will be clipped.

Transitions

Subtitle messages rotate in and out following iOS motion principles. Only the text content transitions — the container stays in place.

Text rotation

When a new message replaces the current one, the outgoing text exits downward while the incoming text enters from above, creating a vertical “ticker” motion.

StepPropertyValueDurationEasing
Exit (outgoing text)opacity1 → 00.25seaseIn
translateY0 → 8px
Enter (incoming text)opacity0 → 10.35sspring
translateY−8px → 0

Sequencing

  • The exit completes before the enter begins — the two never overlap.
  • Follow the iOS pattern: exit with easeIn (0.25s), then enter with spring (0.35s).
  • The leading icon (if present) transitions together with the text as a single unit.

First appearance

  • When the Subtitles component first appears on screen, the text enters with the same spring animation (0.35s, translateY −8px → 0, opacity 0 → 1).
  • No exit animation is needed for the first message.

Reduced motion

  • When prefers-reduced-motion: reduce is active, replace the vertical slide with a simple crossfade (opacity only, 0.2s easeInOut).

Accessibility

  • Use role="status" so screen readers announce the message without interrupting the user.
  • Add aria-live="polite" when the message appears dynamically (e.g., after completing a task).
  • Leading emojis are decorative — the text content should be self-explanatory without the icon.
