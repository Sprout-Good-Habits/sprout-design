Empty States
Patterns for blank screens, first-run experiences, and zero-data states across Sprout. An empty state communicates that a screen, container, or section has no content to display yet — it reassures the user and, when appropriate, guides them toward an action.

When to use

Use an empty state to communicate that a screen or section has no content to display yet and guide users toward action.

  • ✓ When a list, feed, or container has zero items to display.
  • ✓ On first-run screens to guide the user toward their first action.
  • ✓ After a search or filter returns no results.
  • ✓ When a feature is available but requires setup before content appears.

When not to use

  • ✗ Don’t use for loading states — use the Loading Indicator instead.
  • ✗ Don’t use for error or connectivity states — use the Error Screen pattern instead.
  • ✗ Don’t use for transient confirmations — use a Toast.

Anatomy

The empty state component is a vertically stacked container with flexible slots. All elements are centered.

All slots
⚪
Label
Description
  • Icon (required) — 56px TossFace emoji. Sets the emotional tone.
  • Title (required) — text-md bold (16px/24px), --text-primary. Short, clear statement.
  • Description (optional) — text-sm regular (14px/20px), --text-secondary. Extra context or guidance.
  • Primary CTA (optional) — btn-primary btn-lg. Guides the user to take action.
  • Secondary CTA (optional) — btn-secondary btn-lg. Always paired with a primary CTA, never alone.

Types

Empty states appear in three contexts, each with different sizing and emphasis.

Full-screen

Used when an entire page has no content. Centers the empty state vertically and horizontally within the main content area. Includes a large icon, title, description, and optional CTA button.

In-sheet

Used inside bottom sheets or modals. Uses a more compact layout with a smaller icon (48px) and shorter copy. Keeps the sheet height minimal.

In-card

Used inside cards or inline containers. The smallest variant, typically just an icon and a single line of text. Should not include CTA buttons.

Guidelines

  • Be specific. Tell the user exactly what’s missing and, when possible, what action they can take. Avoid generic messages like “Nothing here.”
  • Stay positive. Frame the empty state as an opportunity, not a dead end. Use encouraging language appropriate for the audience (parents vs. kids).
  • Use a relevant icon. Pick a TossFace emoji that relates to the content type (e.g., a camera for an empty photo gallery, a star for an empty favorites list).
  • Limit to one CTA. Full-screen empty states can include a primary action button. Don’t add secondary actions unless there’s a clear alternative path.
  • Don’t use empty states for errors. Connectivity issues, timeouts, and failed loads should use the Error Screen pattern instead.
  • Keep copy short. Title should be one line. Description should be one to two sentences at most.

Content guidelines

  • Title: Short, clear statement. Lead with “No” for absent content (“No messages yet”) or a positive statement for completed states (“All done!”).
  • Description: One sentence of context. Explain what will appear here or how to get started.
  • Tone: Friendly and encouraging — this is a kids app. Avoid technical language.
  • TossFace icon: Pick an emoji that relates to the content type. Examples: 📭 messages, 🌱 goals, 🎒 apps, 🔍 search, 🏆 achievements, ⭐ favorites, 📋 tasks, 👋 friends, ✨ welcome.
  • CTA copy: Use action verbs (“Add a goal”, “Browse apps”, “Get started”). Keep it under 3 words when possible.

Accessibility

  • Use role="status" or aria-live="polite" for empty states that appear dynamically (e.g. after filtering).
  • TossFace icons should have aria-hidden="true" — the title and description convey the meaning.
  • Disabled buttons must include aria-disabled="true".
  • Ensure title and description text meet WCAG AA contrast (4.5:1). The documented token pairs (--text-primary on --bg-primary) satisfy this.
