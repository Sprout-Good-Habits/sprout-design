Match Pairs
An exercise pattern where the kid matches items from two columns by tapping one on each side. Uses tile-style cards with validation states for feedback. The composer is hidden and a disabled "Check" button takes its place, enabling once all pairs are matched.
  • ✓ Use for vocabulary, associations, cause-and-effect, or any pairing exercise.
  • ✓ Use when you want the kid to learn through trial and error (wrong answers are safe).
  • ✗ Don't use for more than 5 pairs — the columns become too long.
  • ✗ Don't use when there's only one correct answer — use single-select tiles instead.

Behavior

  • Select: Tap a tile on either side to select it (blue highlight). Tap another on the same side to switch. Tap the same tile again to deselect.
  • Auto-check: When one tile from each column is selected, the pair is checked automatically after 300ms.
  • Correct pair: Both tiles flash green (success) for 800ms, then become disabled (flat, muted text, leading in luminosity).
  • Wrong pair: Both tiles flash yellow (warning) for 300ms, then transition back to default. The kid can try again.
  • Check button: Stays disabled (gray) until all pairs are matched. Kid taps "Check" to finish the exercise.
  • Composer: Hidden during the exercise. Returns after the kid taps "Check".

Edge cases

  • Composer hidden: The composer is replaced by the "Check" button during the exercise. The kid cannot type, send images, or record audio.
  • Same-side taps: Tapping two tiles on the same side just switches the selection. Only cross-column pairs trigger a check.
  • Rapid tapping: While a pair is being validated (success or warning animation), all tiles are locked (processing flag) to prevent double-checks.
  • Retry: Wrong pairs return to default — the kid can try as many times as needed. There is no penalty.
  • Max pairs: Keep to 3–5 pairs. More than 5 makes the columns too long for the chat thread.
  • Content types: Each tile can contain emoji + text, text only, or emoji only. Left and right columns don't need to match format.

Accessibility

  • Each tile is focusable and activatable with keyboard.
  • Wrong answers use a non-punitive yellow flash — no error sounds or negative language.
  • Disabled tiles remove pointer-events and mute visually with luminosity blend on emojis.
  • The "Check" button has aria-disabled when not all pairs are matched.
