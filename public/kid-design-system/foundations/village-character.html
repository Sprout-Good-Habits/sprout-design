Village Character
The Village Character is the animated avatar that represents every family member in Sprout. Built as a Rive animation with deep customization — skin tone, hairstyle, hair color, beard, clothing — it’s a person’s face in the product. The character appears everywhere: profiles, challenges, chat, coaching sessions, and the family view.

Design principles

Identity, not decoration

The avatar shows how you relate to someone, not just who they are. Every village member gets a character that represents them everywhere in the product.

Always grounded

The bottom edge of the container always clips the character. It is never floating in empty space. Whether in a 24px circle or a full-width banner, the character is anchored.

Consistent across contexts

The same Rive file and state machine powers every instance — from tiny avatars to profile banners. Customization choices persist everywhere.

User-owned customization

Background colors, hairstyles, skin tones, and clothing are chosen by the user in the character customizer. Never hard-code these values.

Anatomy

  • Rive file — character2.91.riv. Contains all body parts, customization options, and animations in a single state machine.
  • Artboard — Village-character. The specific artboard within the Rive file.
  • State machine — State Machine 1. Controls idle behavior, mouth animations, and customization inputs.
  • Fit mode — rive.Fit.Contain. The character scales to fill the canvas while preserving aspect ratio.
  • Alignment — rive.Alignment.Center for avatars, varies by context for larger placements.

Rendering guidelines

Avatar circles (XS–XL)
In small circles, only the head and upper body are visible. The canvas extends beyond the circle bounds and overflow: hidden clips to the circle. Position with negative top offset to frame the face. Use rive.Alignment.Center.
Profile banners
In full-width banners, the character is centered horizontally with the upper body visible. The bottom of the container clips the torso. Canvas is sized to show head through mid-chest. Use rive.Alignment.Center.
Character customizer
In the customizer, the full character is visible on the sky background. The character sits above the customizer panel. Canvas is large enough to show the complete figure. Use rive.Alignment.Center.

When to use

  • ✓ Use the Village Character whenever displaying a user’s identity — profiles, chat bubbles, participant lists, family view.
  • ✓ Always render from the Rive file (character2.91.riv) — never assemble from static images.
  • ✓ Always respect the user’s saved customization (skin, hair, clothing, background).
  • ✓ Fall back to Silhouette before the user has customized, or User initial when no data is available.
  • ✗ Don’t use the Village Character for the Sprout mascot — Sprout has its own Rive file (sprot2.97_.riv). See Character (Sprout).
  • ✗ Don’t hard-code customization values — always read from the user’s saved state.
  • ✗ Don’t use on content-heavy screens where the character would obstruct the UI.

Accessibility

  • Motion: Respect prefers-reduced-motion. Pause or disable Rive animations when the user prefers reduced motion.
  • Decorative in context: When the character appears inside a labeled component (Avatar Row label, Toolbar title), the canvas is decorative — the label provides the accessible name.
  • Standalone contexts: When the character is the primary content (profile banner), add role="img" and aria-label describing the user.
  • Performance: Lazy-initialize Rive instances only when visible. Pause off-screen instances to save resources.
