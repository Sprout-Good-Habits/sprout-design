Video
Full-screen video recording with a character guiding the kid through the session. Uses a floating conversational feedback card and a dedicated Button Recording control paired with two Button Utility buttons for camera flip and flash. No live elapsed-time timer: the Button Recording state (Start or Stop) is the only recording indicator.
  • ✓ Use the Video pattern for continuous recording where Sprout encourages the kid while they perform or demonstrate something.
  • ✗ Don’t use for still photos. Use the Camera pattern instead.

When to use

Use the Video pattern whenever the kid needs to record a continuous session and a character should stay present to guide them throughout.

  • ✓ Habit proofs that require a demonstration (reading aloud, piano practice, stretching).
  • ✓ Practice sessions where the character narrates or supports each rep.
  • ✓ Short self-recorded messages to a parent or mentor.

When not to use

  • ✗ Single-shot photo capture. Use the Camera pattern.
  • ✗ Long-form video without coaching. Use a plain system recorder.
  • ✗ Live video calls between two people. Needs a different component tree.

Anatomy

The Video pattern mirrors the Camera pattern layer-for-layer so kids can move between the two without re-learning the layout. Heights assume a 402×874 iPhone mockup. Layer order from bottom to top:

Toolbar top:62 Conversational Feedback top:126 Bottom bar bottom:34 Home indicator bottom:8
0. Viewfinder (full bleed, edge-to-edge): The live camera feed fills the entire device screen. In docs it is shown as a neutral #d9d9d9 placeholder.
1. Status bar (62px): System chrome passed through from the device.
2. Toolbar (top 62px): A Toolbar component composing a Button Utility close. Three variants are valid here, all built on the Button Utility slot pattern. Pick whichever fits the host screen:
  • toolbar-quest-transparent (Quest Overlay): transparent background with a btn-util close and a Progress Bar. Best default for a full-bleed recording view.
  • toolbar-default: leading slot holds a btn-util close, optional absolute centered title, optional trailing slot for a primary action. Use when the recording is embedded inside a labelled flow.
  • toolbar-title-2-line: leading and trailing btn-util slots with a centered two-line title. Use when the parent flow needs a title plus subtitle above the viewfinder.
The demo above uses toolbar-quest-transparent. Never use toolbar-quest (the non-overlay variant): its sky-blue background clashes with the viewfinder. The Progress Bar represents overall session progress (not elapsed recording time).
3. Conversational feedback (top 126px, inset 16px): A Conversational Feedback component in the Conversational type, Incoming direction. Uses an avatar-portrait and subtitles-overlay so the bubble reads clearly over the viewfinder. The character is either Sprout (.rive-sprout) or the Village Character (.rive-user) depending on who is guiding the kid.
4. Bottom bar (72px, pinned 34px from bottom): Flip camera (48×48 Button Utility), Button Recording Secondary Stop (72×72, red square), and Flash toggle (48×48). In the Ready state the center button swaps to Secondary Start (white with a neutral indicator dot). Swap to Primary Start / Primary Stop only if the recording needs the stronger brand fill.
5. Home indicator (134×5, pinned 8px from bottom).

Composition

Every element on screen is an existing component. The pattern owns only the layout and the state transitions.

Toolbar
Quest Overlay variant with a Button Utility close and a nested Progress Bar. See Anatomy for the full list of valid toolbar variants.
⭐
🎯
Button Utility
48×48 icon button used three times: Close in the Toolbar, Flip camera, and Flash toggle. Icons are inline SVG (camera-rotate and zap).
Progress Bar
Nested inside the Toolbar. Represents the kid’s progress through a multi-step quest, never elapsed recording time.
Conversational Feedback
Conversational type, Incoming direction. Pairs an avatar-portrait with subtitles-overlay so the bubble reads against the viewfinder. Copy can rotate on a steady cadence to keep the kid oriented.
Button Recording
Swaps between Secondary Start (white with a neutral indicator dot, pre-record) and Secondary Stop (red square, mid-record) as the state changes. Swap to the Primary variants only if the recording needs the stronger brand fill.

Character’s role

The guiding character (either Sprout or the Village Character) stays present through the session. Copy tone should match the host flow: neutral and instructional for utility captures, warmer and more supportive for practice or celebration flows. Rotate lines on a steady cadence so the kid feels oriented without being distracted.

  • Ready state: “Ready when you are. Tap the shutter to start.”
  • Recording, neutral: “I’m listening. Take your time.”
  • Recording, supportive: “Keep going. You’re almost there.”
  • ✓ Match the host flow’s tone. The pattern is neutral by default and only reads as positive when the host flow calls for it.
  • ✓ Keep every line under 14 words so it fits the feedback bubble.
  • ✗ Don’t coach technique (“Slow down”, “Speak up”). That’s a parent’s job, not the character’s.
  • ✗ Don’t force encouragement. A utility recording flow should sound like instructions, not a pep talk.

States

The Video pattern has two top-level states. The bottom bar and progress bar update together so the kid always reads the same signal.

  • Ready: Viewfinder live, Button Recording in Secondary Start (white with a neutral indicator dot). The guiding character prompts the kid to start.
  • Recording: Button Recording in Secondary Stop (red square). The guiding character rotates its copy on a steady cadence. No elapsed-time indicator is shown: the Stop button alone signals that recording is active.

Accessibility

  • Live region: Wrap the Conversational Feedback in role="status" aria-live="polite" so Sprout’s rotating encouragement is announced to screen reader users.
  • Recording toggle: The Button Recording uses aria-label="Start recording" in Ready and aria-label="Stop recording" with aria-pressed="true" during Recording. Screen readers announce the state change on toggle.
  • Progress bar: The Toolbar’s Progress Bar represents overall session progress, not elapsed recording time. Use role="progressbar" with aria-valuenow / aria-valuemin / aria-valuemax so assistive tech can describe how far into the session the kid is.
  • Reduced motion: Rive animation respects prefers-reduced-motion. Never add parallax or viewfinder effects.
