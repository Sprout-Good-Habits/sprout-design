Response Tiles Multiple
Tiles with checkboxes that let the kid pick multiple options before confirming. A primary button at the bottom activates once at least one tile is selected. Uses the DS Tile component (Checkbox variant).
  • ✓ Use when the question allows multiple answers (e.g. "What did you do today?").
  • ✓ Use when the kid should review their choices before committing.
  • ✗ Don't use for single-choice questions — use Single-select tiles instead.
  • ✗ Don't use when the number of options exceeds 5 — consider a different pattern.

Tap tiles to toggle selection. The confirm button activates once at least one tile is checked. Tap "Done" to confirm.

Behavior

  • Toggle: Each tile toggles independently. Tap to check, tap again to uncheck.
  • Confirm button: Starts disabled (gray). Activates as primary blue when ≥1 tile is checked. Shows a count badge.
  • Submit: Tapping the confirm button dismisses the group. All selected labels combine into one outgoing message.
  • Message format: Selections are joined naturally — "I did school, sports and art today!"

Edge cases

  • Composer replaced: Unlike single-select tiles (which are suggestions), multi-select hides the composer entirely. The confirm button takes its place at the bottom of the screen, always anchored. The kid must make their selections and tap the confirm button. Once confirmed, the composer returns.
  • No skip: There is no way to dismiss tiles without selecting. If the question is optional, add a "Skip" tile as the last option.
  • All deselected: If the kid unchecks all tiles after checking some, the confirm button returns to disabled state. The count badge shows 0.
  • Max selections: No hard limit, but the message format degrades past 4 selections.
  • 6+ options — scrollable: When there are 6 or more tiles, the tile list renders inside a scrollable container (max-height 340px) with a fade hint at the bottom. The confirm button stays sticky below the scroll area, always visible. A thin scrollbar and bottom fade indicate more content.
  • Scroll fade: The bottom fade disappears once the kid scrolls to the end, so the last tile is fully visible.

Content guidelines

  • Sprout's question should explicitly say "pick all" or "as many as you want" so the kid knows they can select multiple.
  • Confirm button label should be short — "Done" is the default. Use "Next" or "Send" if it fits the context better.
  • Keep the total to 3–5 tiles. More than 5 options pushes the confirm button out of view.

Accessibility

  • Container has role="listbox" and aria-multiselectable="true".
  • Each tile has role="option" with aria-checked toggling on tap.
  • Confirm button has aria-disabled when no tiles are selected.
  • Count badge updates live so screen readers can track selection count.
