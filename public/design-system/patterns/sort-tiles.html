Sort Tiles
An exercise pattern where the kid drags tiles to put them in the correct order. Uses the DS Tile Sortable variant with a drag handle. The composer is hidden and a "Check" button takes its place, validating the order on tap.
  • ✓ Use for sequencing, ranking, or ordering exercises (e.g. "smallest to biggest", "first to last").
  • ✓ Use when learning the correct order is the goal — wrong answers are safe to retry.
  • ✗ Don't use for more than 6 items — dragging becomes difficult in a scrollable list.
  • ✗ Don't use when the order doesn't matter — use Multi-select tiles instead.

Behavior

  • Drag: Press and hold a tile, then drag to reorder. A dashed placeholder shows the drop position. The tile elevates with a shadow while dragging.
  • Number badges: Each tile shows its current position number (1, 2, 3...). Numbers update automatically when tiles are reordered.
  • Check: Tap "Check" to validate the order. Each tile gets a state:
    • Success (green) — correct position
    • Warning (yellow) — wrong position, flashes then returns to default for retry
  • All correct: Button changes to "Correct!", exercise dismisses after 800ms, composer returns.
  • Some wrong: Wrong tiles flash yellow for 500ms then return to default. The kid can drag and try again.
  • Composer: Hidden during the exercise. Returns after completion.

Edge cases

  • Composer hidden: The composer is replaced by the "Check" button during the exercise. The kid cannot type, send images, or record audio.
  • Rapid tapping: While validation is showing (success/warning flash), tiles are locked to prevent re-dragging.
  • Partial correct: Tiles in the right position show green, tiles in the wrong position show yellow. The kid only needs to fix the yellow ones.
  • Retry: After wrong positions flash yellow, all tiles return to default and the kid can drag again. No penalty.
  • Max items: Keep to 3–6 items. Longer lists make dragging tedious on mobile.
  • Shuffled start: Items are always shuffled on load. The shuffle guarantees the initial order is never correct.

Accessibility

  • Tiles use touch-action: none and pointer events for cross-device drag support.
  • Wrong answers use non-punitive yellow flash — no error sounds or negative language.
  • Number badges provide clear position feedback for screen readers.
  • The "Check" button confirms the order — dragging alone doesn't auto-validate.
