Response Tiles Single
Tiles presented inside the chat thread as tappable suggested responses. These are suggestions, not requirements — the kid can pick a tile or type their own response instead. Uses the DS Tile component (None variant) to keep a consistent visual language.
  • ✓ Use when Sprout asks a question with 2–4 distinct answer options.
  • ✓ Use to guide the conversation without requiring the kid to type.
  • ✗ Don't use for open-ended prompts — use the composer instead.
  • ✗ Don't use when more than one answer can be selected — use Multi-select instead.

Behavior

  • Appear: Tiles slide in below Sprout's message with a staggered animation (50ms delay per tile).
  • Select: Tap a tile — it highlights with the .checked state (blue bg, brand border). After 400ms the group fades out.
  • Dismiss: The group collapses and the kid's selection appears as an outgoing chat bubble.
  • Chain: Sprout can respond with a follow-up message and present a new tile group.

Edge cases

  • Composer stays open: Tiles are suggestions, not mandatory. The kid can ignore the tiles and type their own response, send a photo, or record audio instead. If they send a message, the tile group dismisses automatically.
  • Single tap: Selection is immediate — one tap selects and triggers the dismiss animation. There's no undo once a tile is tapped.
  • Chained tiles: A new tile group can appear after the previous one dismisses.
  • Free text override: If the kid types and sends a message while tiles are showing, the tiles dismiss and the typed message is used instead.

Content guidelines

  • Keep labels short — 1–3 words. Use the description for context if needed.
  • Always include a leading emoji to make options scannable at a glance.
  • Limit to 2–4 tiles per group. More options overwhelm the chat flow.
  • Labels should feel like things the kid would say, not formal categories.

Accessibility

  • Container has role="listbox", each tile has role="option" with aria-selected.
  • Tile touch target is minimum 56x44px (DS Tile default).
  • Reduced-motion users see tiles without slide-in animation.
