Result Screen
Full-screen pattern for quest and task outcomes. Communicates completion status to the kid with an encouraging tone, a large Sprout character, and a single Continue action. Used for approved completions, partial completions, and pending parent review.
Result Screen
Quest Complete!
You practiced for 10 minutes and played it 5 times. I'm so proud of you!

When to use

Use a result screen to communicate the outcome of a completed quest or task to the kid.

  • ✓ After a quest reaches readiness = 1.0 (full completion).
  • ✓ After a quest is auto-approved above the parent's threshold (partial completion).
  • ✓ After a quest is sent to parent for review (below threshold or expired).
  • ✓ Any kid-facing end state that needs to feel warm and encouraging.

When not to use

  • ✗ Don't use for error states — use the Error Screen pattern.
  • ✗ Don't use for parent-facing review screens — those use the Parent Review layout.
  • ✗ Don't use for in-progress states (readiness bar, camera view).

Anatomy

The result screen is a full-screen absolutely-positioned layout with five layers.

  • Background (required) — var(--bg-sky) solid fill. Same across all variants.
  • Title + Description (required) — Positioned at top: 100px, centered. Title is 30px bold in --text-primary. Description is 18px regular in --text-tertiary, max-width 370px.
  • Sprout Character (required) — Rive canvas, 690×690 at bottom: -196px. Uses Fit.Contain, Alignment.BottomCenter. Character must be grounded — clipped by the bottom edge, never floating.
  • Grass (required) — Two layers at the bottom: 24px rounded top edge + 89px solid fill, both #86cb3c.
  • Continue button (required) — btn-sprout btn-primary btn-lg, full-width at bottom: 45px.
  • Home indicator (required) — Standard 134×5px black pill at bottom: 8px.

Sprout behavior

Sprout's expression and speech are driven by the Rive state machine and vary by outcome.

VariantRive triggerMouth animationTone
Full Completion celebration Speaks the description text (1700ms delay after celebration) Celebratory, proud
Partial (Above Threshold) celebration Speaks the description text (1700ms delay after celebration) Encouraging, positive
Sent to Parents smile Speaks the description text (200ms delay after smile) Reassuring, warm
  • Mouth animation duration is calculated from the description word count: max(1500ms, wordCount × 250ms).
  • A hidden .subtitle-card p element holds the text that drives the mouth animation timing.
  • The celebration trigger must complete before the mouth starts — the 1700ms delay ensures this.
  • Never show a sad or negative Sprout expression on result screens. Even “Sent to Parents” uses smile for an encouraging tone.

Copy guidelines

The description text is what Sprout “says” to the kid. Write it in first person from Sprout’s voice.

  • Full Completion — Celebrate the specific achievement. Mention what they did and how well. “I'm so proud of you!”
  • Partial — Acknowledge effort, note what was done, encourage next time. “So close!”
  • Sent to Parents — Explain what happened without blame. Use “we” language. “We'll hear back from them soon!”
  • Keep descriptions under 25 words for comfortable mouth animation timing.
  • Never frame the outcome as failure. Every result screen is a positive moment.

Adding a new variant

When creating a new quest outcome screen (e.g. “Redo Requested” or “Expired”), follow this checklist:

  1. Duplicate an existing result screen HTML block.
  2. Update the title and description text.
  3. Choose a Rive trigger that matches the emotional tone (celebration, smile, characterwave).
  4. Add data-rive="triggerName" to the nav item.
  5. Add a hidden .subtitle-card p with the description text for mouth animation.
  6. Keep the background as var(--bg-sky) — don’t use gradients or custom colors.
  7. Keep the layout structure identical. Only the content changes.

Rive character

The Sprout character uses sprot2.97.riv with state machine State Machine 1.

  • File: sprot2.97.riv — hosted at https://sprout-hq.vercel.app/assets/sprot2.97.riv
  • Canvas: 1380×1380 internal resolution, displayed at 690×690 CSS pixels
  • Fit: Fit.Contain, Alignment.BottomCenter
  • Positioning: bottom: -196px, centered horizontally — character is grounded and clipped by the grass edge
  • Default customization: hairID = 1 (short hair)

Available triggers for result screens

  • celebration — Full-body bouncy animation. Use for positive outcomes (completion, approval).
  • smile — Warm encouraging expression. Use for neutral-positive outcomes (sent to parents).
  • characterwave — Wave + smile. Could be used for “see you next time” type outcomes.
