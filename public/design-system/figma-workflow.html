MCP Tools

Four tools from the Figma MCP server power the design-to-code pipeline.

get_design_context

Fetches the structured representation of a Figma node — layout, styles, children, and auto-layout properties. Always run this first to understand the component structure.

get_metadata

Returns a high-level node map of a Figma file or page. Use this when get_design_context returns a response that is too large or truncated, then re-fetch specific nodes.

get_screenshot

Captures a pixel-accurate screenshot of a Figma node. Use as the visual reference to validate your implementation against — aim for 1:1 fidelity.

get_variable_defs

Returns exact token names and resolved color values from the Figma file. Never guess colors from screenshots — always verify with this tool before implementing.

Translation Rules

The MCP server outputs React + Tailwind code. These rules govern how to convert that output into production code for Sprout.

Code

Output is reference, not final code

Treat Figma MCP output as design intent. Convert React + Tailwind to semantic HTML + CSS custom properties from tokens.css. Map every color, size, and spacing value to the nearest Sprout token.

Inspect

What to look for

Check for Code Connect mappings, developer annotations, token references, and raw values. Reuse existing component patterns before inventing new ones. Verify tokens with get_variable_defs.

Assets

Character assets

Never attempt to assemble multi-part SVG characters (body, hair, eyes). Ask the designer to export a single composite SVG from Figma and store it in /brand/assets/.

Verification Checklist

Before marking an implementation complete, confirm every item in this checklist.

✓
Visual fidelity — implementation matches the Figma screenshot at 1:1 accuracy
✓
Token mapping — every color, size, and spacing uses var(--token), no hardcoded values
✓
Component reuse — existing patterns are used before creating new ones
✓
Assets by reference — SVGs and images use <img src="...">, not inline markup
✓
Responsive — layout adapts gracefully at 1024px and 768px breakpoints
✓
Accessibility — semantic HTML, keyboard support, proper contrast, and ARIA labels
