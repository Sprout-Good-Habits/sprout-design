Radius
Border radius tokens and shadow patterns used across the Sprout design system. Radius values are tokenized; shadows follow consistent hardcoded patterns.

Border Radius

Five radius tokens cover every corner in the system. Use these tokens exclusively — never hardcode pixel values for border-radius.

xs 4px --radius-xs
sm 6px --radius-sm
md 8px --radius-md
xl 12px --radius-xl
full 9999px --radius-full

Radius Usage

Which token to reach for in each context.

TokenValueUse for
--radius-xs4pxSmall chips, inline code, tags
--radius-sm6pxNav items, small interactive elements
--radius-md8pxCards, modals, info notes, code blocks
--radius-xl12pxLarge cards, containers, demo panels
--radius-full9999pxPills, avatars, circular buttons
Note: Buttons and text inputs use 16px (--radius-2xl) and sheet top corners use 24px (--radius-4xl). These values exist in the component specs but are not yet part of the core token set shown above.

Shadow Patterns

Shadows are not tokenized — use these hardcoded patterns consistently across all implementations.

Primary 0 4px 0 0 var(--brand-600)
Secondary 0 3px 0 0 var(--gray-200)
Destructive 0 4px 0 0 var(--red-600)
Disabled 0 4px 0 0 var(--gray-400)
Elevation 0 8px 30px rgba(0,0,0,0.1)
Card Hover 0 4px 24px rgba(0,0,0,0.06)

Active State (Press Effect)

On press, the shadow collapses and the element translates down by the shadow’s Y offset. Click or hold these buttons to see the effect.
