Guidelines

Motion principles following Apple Human Interface Guidelines.

1

Spring is the default

iOS uses spring animations as the primary motion curve. Use cubic-bezier(0.5, 1.8, 0.3, 0.8) for anything entering the screen, responding to touches, or changing state. Spring creates natural, physical motion that feels alive. Only fall back to easeInOut for subtle property changes like opacity or color.

2

Respect reduced motion

Honor @media (prefers-reduced-motion: reduce) — the CSS equivalent of UIAccessibility.isReduceMotionEnabled. Replace springs and slides with simple crossfades. Essential state indicators (spinners, progress) should remain but may reduce intensity.

3

Active state pattern

Pressed/active states use box-shadow: none; transform: translateY(Npx) where N matches the resting shadow offset. This creates a physical “push” feel that complements Sprout’s 3D button style. The press transition uses 0.1s easeInOut.

4

Enter with spring, exit with easeIn

Elements enter using spring (fast overshoot, settle into place) and exit with easeIn (gentle start, fast departure). Exits are shorter than enters — e.g., sheet present 0.5s spring vs. dismiss 0.35s easeIn. This matches UIViewController presentation.

5

Motion builds spatial awareness

Transitions should reinforce where things live in the interface. Navigation pushes slide in from the right; sheets rise from the bottom; alerts scale up from center. Consistent spatial relationships help users build a mental map of the app, following iOS's emphasis on depth and hierarchy.
