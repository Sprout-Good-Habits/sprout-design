Avatar Profile
A hero banner at the top of profile screens that prominently displays the user’s village character on a customizable background color. The character is always grounded — the bottom edge of the banner clips the body, keeping the head and upper torso visible. Background colors are chosen by the user in the character customizer.
  • ✓ Use at the top of profile pages to give the user a strong sense of identity.
  • ✓ Always respect the user’s chosen background color — never hard-code it.
  • ✓ Pair with a Toolbar overlay for navigation (back) and actions (settings).
  • ✓ The character is always grounded — clipped at the bottom, never floating.
  • ✗ Don’t use for non-profile contexts — this is specifically for user identity.
  • ✗ Don’t add text or overlays on top of the character — keep the banner focused on the avatar.

Anatomy

Container (full-width, 264px tall)
Background color (user-selected)
Rive character (centered, grounded)
  • Container — Full-width block, 264px tall, with overflow: hidden to clip the character at the bottom.
  • Background color — Defaults to --bg-sky (#e0f2fe). Users choose from 10 pastel options in the character customizer.
  • Rive character — Centered horizontally, positioned so the head and upper torso are visible. Uses the Village-character artboard with State Machine 1.
  • Always grounded — The bottom edge of the container always clips the character. It is never floating in empty space.
  • Toolbar (optional) — Overlays the top of the banner with transparent background. Contains back button, title (user name), and trailing action (settings). Uses Button Utility for interactive controls.

Accessibility

  • The banner container should have role="img" and aria-label describing the user (e.g., "Sam Lee's profile avatar").
  • The Rive <canvas> is decorative — it should have pointer-events: none and no focus.
  • When paired with a Toolbar, the toolbar buttons provide their own accessible labels (e.g., "Back", "Settings").
  • Background color choice is cosmetic and does not convey meaning — no ARIA attributes are needed for the color.
