List Item
A flat, borderless row for displaying content in lists. Unlike Tiles (which are bordered, tappable cards for selection flows), List Items are lightweight rows for content lists like friends, family members, courses, and search results.
  • ✓ Use a List Item for flat content lists — friends, members, courses, search results.
  • ✗ Don’t use a List Item for selection flows where each option is a distinct card — use a Tile instead.

Usage guidelines

List Item vs Tile

  • ✓ Use List Item for flat content lists where items are not individually selectable cards — friends, members, courses, search results.
  • ✓ Use Tile when each option is a distinct, tappable card with a border and shadow — selection flows, exercises, onboarding.
  • ✗ Don’t use a List Item inside a card or tile — keep the hierarchy flat.

Trailing types

  • ✓ Use Checkbox for multi-select lists — adding multiple friends, selecting contacts.
  • ✓ Use Radio for single-select from a flat list.
  • ✓ Use Chevron when tapping the row navigates to a detail screen.
  • ✓ Use Metadata for displaying a secondary value like XP, dates, or counts.
  • ✓ Use Button for inline actions — ADD, INVITE, NUDGE.
  • ✓ Use None when no trailing action is needed.
  • ✗ Don’t mix different trailing types in the same list section.

Leading element

  • ✓ Use Avatar when the row represents a person.
  • ✓ Use Icon for category or feature lists — flags, app icons, action icons.
  • ✓ Omit the leading element for simple text lists — username suggestions, settings.
  • ✗ Don’t mix different leading types in the same list section.

Anatomy

A list item row is composed of up to four parts. All parts are optional and configurable.

  • Leading (optional) — 40px. Avatar (circle with Rive character) or Icon (Toss Face emoji).
  • Label — Bold, text-md (16px/24px). The primary text.
  • Description (optional) — Regular, text-sm (14px/20px), text-tertiary color. Can include a leading icon (emoji).
  • Trailing (optional) — Checkbox (24px), Radio (24px), Chevron (24px), Metadata (text), or Button (btn-xs).

Accessibility

  • Role: Wrap items in a role="list" container. Each row uses role="listitem". Checkbox/Radio trailing types should use aria-checked.
  • Keyboard: Checkbox and Radio items must be togglable with Enter or Space. Chevron items should activate on Enter. Button trailing uses standard <button> focus.
  • Touch target: The full row is at least 48px tall, exceeding the 44×44px guideline.
  • Contrast: Label (--text-primary) and description (--text-tertiary) both pass WCAG AA on white backgrounds.
