Navigation item

Navigation item gives users a rich way to choose between options and navigate to another screen or step in a flow.

  • ✓ If you want to give the user a rich way to make a choice and navigate to another screen or step in a flow.
  • ✓ If making a choice is the primary action on the screen.
  • ✓ If there are multiple options that the user can choose from.
  • ✓ If some extra context, such as an icon, would help the user understand where they’ll navigate to if they select that option.
  • ✗ To ask the user to choose options within a form — use radio or checkbox instead.
  • ✗ When there is just one primary option, and perhaps an alternative secondary option — use a button instead.
  • ✗ When the user needs to complete all the options on a screen, rather than choose one.

Checkbox item

Checkbox item allows users to select multiple items from a mutually exclusive list of options.

  • ✓ Within a select input.
  • ✓ When a user needs to choose within a list of more than 3 options.
  • ✗ Directly in a form, unless used as a full screen.
  • ✗ As a standalone or in place of a switch.

Radio item

Radio item allows users to select a single item from a mutually exclusive list of options.

  • ✓ To select a single option within a list of options.
  • ✓ Within a select input.
  • ✓ When a user needs to choose within a list of more than 3 options.
  • ✗ In a radio group.
  • ✗ Directly in a form, unless used as a full screen.
  • ✗ As a standalone or in place of a switch.

Icons

Icons are optional and are used as a visual aid to the list item.

  • ✓ Use a relevant Toss Face icon to support content.
  • ✗ Don’t use an illustration in place of the icon.

Anatomy

A list item is composed of up to four parts. Only the label is required; the icon, description, and trailing indicator are all optional.

Content guidelines

List item copy should:

  • Keep labels short — ideally 2–4 words
  • Start with a verb or noun that names the option
  • Use sentence case (only capitalise the first letter of the first word)
  • Use the description for extra context only when the label isn’t self-explanatory
  • Avoid ending labels or descriptions with a period

Accessibility

List items are interactive selection controls. They must be usable by all input methods.

  • Role: Checkbox type → use role="option" in a role="listbox", or native <input type="checkbox">. Radio type → use role="radio" in a role="radiogroup", or native <input type="radio">. Navigation type → use <a> or role="link".
  • Keyboard: Items must be selectable with Enter or Space. Arrow keys should move focus between items.
  • Focus ring: A visible 2px focus ring appears on keyboard navigation. Do not remove :focus-visible outlines.
  • ARIA state: Use aria-selected="true" for checked items and aria-selected="false" for unchecked. Radio groups should use aria-checked.
  • Contrast: Labels (--text-primary on --bg-primary) and descriptions (--text-secondary on --bg-primary) both pass WCAG AA.
  • Touch targets: The entire row (minimum 56px tall) is the tap target, exceeding the 44×44px guideline.
