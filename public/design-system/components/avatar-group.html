Avatar Group
Avatar groups display a cluster of overlapping circular Avatars representing users and characters in a conversation or shared context. They are used inside a Button Utility in the Toolbar trailing slot.
  • ✓ Use in the Chat toolbar trailing slot to show who is in the conversation.
  • ✓ Use inside a Button Utility so users can tap the group to see participant details.
  • ✓ Always include the Sprout character avatar when Sprout is present in the chat.
  • ✗ Don’t use avatars standalone outside of their Button Utility container.
  • ✗ Don’t mix avatar variants within a group — use either village characters or initials consistently.

  • Avatar circle — XS (24px), SM (32px), or MD (40px) circle with 2px white border, stacked with proportional overlap.
  • User avatar — An Avatar displaying the user’s customized village character (or first-name initial fallback) with their chosen background color.
  • Sprout avatar — Displays the Sprout mascot via Rive.
  • Overflow counter — When more than 6 members, the 6th circle shows “+X” with the remaining count.
  • Container — Flex row with align-items: center and right padding for visual balance inside the Button Utility.

Individual avatar

Each circle in the group is an Avatar component. Users customize their village character with hairstyles, skin tones, clothing, and background colors. See the Avatar documentation for full details on variants, sizes, customization options, and Rive rendering.

Accessibility

  • The Sprout avatar <img> must always have alt="Sprout".
  • User placeholder avatars should have aria-label describing the user (e.g., aria-label="Mom") once character identities are assigned.
  • The wrapping Button Utility must have an aria-label like "View participants" to describe the tap action.
  • The avatar group itself is decorative — interaction comes from the Button Utility container.
