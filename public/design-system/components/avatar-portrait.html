Avatar Portrait
A portrait-oriented rectangular avatar that reveals more of the character than the circular Avatar. Used inside Conversational Feedback to give the character more presence and personality. The background is always sky blue.
  • ✓ Use inside Conversational Feedback.
  • ✓ Use the Avatar variant when the user has a customized character.
  • ✓ Use the Sprout variant for the Sprout mascot.
  • ✓ The character must always be grounded — clipped by the bottom edge, never floating.
  • ✗ Don’t change the background color — Avatar Portrait always uses --bg-sky.
  • ✗ Don’t use as a standalone identity indicator — use the circular Avatar instead.

When to use

Avatar Portrait gives the character more visual presence in contexts where a small circular avatar wouldn’t have enough impact.

  • Conversational Feedback — Paired with conversational or feedback messages (e.g., streak celebrations, encouragement) to show the character with more presence and personality.

When not to use

  • Don’t use Avatar Portrait for identity display (profiles, participant lists, toolbars) — use the circular Avatar instead.
  • Don’t use as a button — wrap in a Button Utility if it needs to be interactive.
  • Don’t use at sizes other than 120 × 176px — the canvas and clipping are tuned for this size.

Anatomy

Rectangle container (120 × 176px)
Sky blue background (--bg-sky)
Rive character or silhouette SVG
Overflow clipping (character grounded)
  • Rectangle container — 120 × 176px with border-radius: 24px (--radius-xl). Clips all content via overflow: hidden.
  • Background — Always --bg-sky (#e0f2fe). Unlike the circular Avatar, this component does not support custom background colors.
  • Rive character (Avatar & Sprout) — The village character canvas is sized larger than the container and offset upward so the head and upper body are visible, with the lower body clipped.
  • Always grounded — The bottom edge of the container clips the character. The character is never floating in empty space.
  • Silhouette SVG — Anchored bottom-center as a fallback.

Accessibility

  • The container must have aria-label describing the character (e.g., aria-label="Mom's character portrait").
  • When used inside an interactive parent (Message Bubble with tap action), the parent provides the accessible label — mark the portrait as aria-hidden="true".
  • Rive <canvas> elements have pointer-events: none to prevent capturing touch events.
