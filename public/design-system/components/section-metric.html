Section Metric
A compact horizontal row of three metrics that summarise how a child is doing in a given section. Each metric evaluates a different dimension of performance: consistency (how regularly they show up), effort (how much time they put in), and mastery (how their skills are growing). This is the primary way Sprout evaluates kids' tasks.
  • ✓ Use to show a child's progress snapshot across consistency, effort, and mastery.
  • ✓ Always display all three metrics together as a complete set.
  • ✓ Use TossFace emoji for icons: 🔥 (fire) for consistency, ⚡️ (lightning) for effort, 🏅 (medal) for mastery.
  • ✗ Don't display fewer than three metrics or change the metric types.
  • ✗ Don't use as an interactive element. It is display-only.
  • ✗ Don't swap the emoji icons for different glyphs.

The three metrics

Section Metric always shows exactly three dimensions. Together they give parents and kids a balanced view of performance, not just "did you get it right" but "did you show up, try hard, and grow."

🔥 Consistency

Tracks how many consecutive days the child has engaged with a section. The value is a whole number representing streak days. This rewards regular practice, even if individual sessions are short.

Anatomy

The component is a bordered card containing three equally-spaced metric items in a horizontal row. Each item pairs an emoji icon with a stacked value and label.

  1. Container, .section-metric, white background with 2px --border-secondary border and --radius-2xl (16px) corners.
  2. Metric item, .section-metric-item, flex row pairing icon + value, takes equal space via flex: 1 1 0.
  3. Icon, .section-metric-icon, 40px square TossFace emoji at 32px font size.
  4. Value, .section-metric-number, bold 18px primary text showing the metric number.
  5. Label, .section-metric-label, regular 12px secondary text naming the metric.

Content guidelines

  • Consistency values are always whole numbers followed by "Days" (or "Day" for 1).
  • Effort values use a compact duration format: "5m", "20m", "1h". The label is always "Hustle".
  • Mastery values are always a signed percentage: "+10%", "+25%". The label is always "Skills".
  • Keep all values short enough to avoid truncation in the metric item's available space.
  • The emoji icon for each metric is fixed and should never change: 🔥 ⚡️ 🏅.

Accessibility

  • Semantics: Wrap the component in role="group" with an aria-label describing the context, for example aria-label="Math section progress". Each metric item should have its own aria-label combining the dimension and value, e.g. aria-label="Consistency, 7 days".
  • Icons: The emoji icons are decorative. Mark them with aria-hidden="true" so screen readers rely on the value and label text instead.
  • Keyboard: Section Metric is non-interactive, so it does not appear in the tab order.
  • Contrast: The bold value text uses --text-primary on white, and the label uses --text-secondary on white, both meeting WCAG AA requirements.
