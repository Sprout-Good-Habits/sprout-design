Screen Time Card
A status card on the kid home carousel that communicates the current screen time state and lets kids unlock screen time by spending gems.
  • ✓ Use the screen time card to show whether screen time is locked, available, active, or restricted.
  • ✓ Use it inside the kid home carousel alongside task and quest cards.
  • ✗ Don't use it outside the kid home, it is tightly coupled to the screen time state machine.
  • ✗ Don't nest interactive controls inside the card. Tapping the whole card is the only interaction.

Variants

The screen time card has 9 display states derived from a combination of server state, native device state, gem balance, and parent settings. Only two states (Unlock Available and Free Unlock) are tappable.

Unlocked states

These states indicate screen time is currently available or actively being used.

  • Timed Unlock: the title shows a live MM:SS countdown. The progress ring drains as time elapses. The app indicator row shows the currently active app (e.g. YouTube with a coloured icon). Not tappable.
  • Free Unlock: screen time is open with no time limit. Full progress ring, pause icon, app indicator row. Tappable (navigates to screen time detail). Visually identical to Timed Unlock except the title reads "Screen Time is Open" instead of a countdown.
  • Unlock Available: the kid has gems and daily budget remaining. Budget ring, play icon. Title shows the remaining time available for the day in MM:SS format. Tappable (opens the unlock sheet to spend gems).

  • Blocked: the kid must complete remaining habits. Lock icon centered in a segmented ring where the number of segments equals the number of tasks. Completed segments use --sprout-400, incomplete segments use the white 20% track. Title shows "N habits left".
  • Schedule Locked: a parent-configured schedule is blocking access. Lock icon with either a budget ring (if the kid already unlocked screen time earlier today) or a segmented ring (if the kid hasn't unlocked yet and still has habits to complete). Never uses a plain dark circle.
  • Parent Locked: a parent has manually locked screen time. Lock icon with a budget ring showing remaining daily budget.
  • Request Pending: the kid sent an unlock request, waiting for parent approval. Hourglass icon with either a budget ring (if the kid already used screen time today) or a segmented ring (if the kid hasn't finished habits yet).
  • Limit Spent: the daily gem budget is fully used up. Lock icon with an empty ring (track only, no progress fill).
  • No Gems: the kid used their screen time and has no gems left to purchase more. Gem icon with a near-empty budget ring reflecting that most screen time has been consumed.

State Derivation

The display state is computed from multiple inputs: native device screen time state, server lock status, gem balance, daily budget, pending requests, and loading status. States are evaluated in priority order; the first match wins.

  • When a timed_unlock countdown reaches zero, the card briefly shows the loading state while it re-fetches server state.
  • The server state is authoritative over the native module (native may default to "unlocked" on simulators where FamilyControls is unavailable).

Anatomy

Every variant shares the same vertical card layout. The icon area at the top varies by state, and the content below always follows a label-title-subtitle stack.

Screentime Ready to Unlock Spend gems for screen time
  1. Card container (.st-card): sprout-500 green background, 24px radius, 16px padding, vertical flex with 8px gap. 370px wide.
  2. Icon area (.st-icon-area): 80x80 circle containing an SVG progress ring and a center icon. The ring track is white at 20% opacity; the progress fill uses sprout-300. The fill amount represents remaining screen time budget.
  3. Center icon (.st-center-icon): play triangle, pause bars, lock emoji, hourglass emoji, or gem emoji depending on state.
  4. Label (.st-label): always reads "Screentime". 16px regular, white at 92% opacity.
  5. Title (.st-title): primary information. 30px bold display font, pure white. Content varies: countdown, status text, or habit count.
  6. Subtitle (.st-subtitle): supporting context. 16px regular, white at 56% opacity. Optional (absent in loading state).
  7. App indicator (.st-app-row): present in Timed Unlock and Free Unlock. Shows a 24px coloured app icon and the name of the app the kid is currently using.

Progress Ring

The progress ring represents the kid's remaining screen time budget. As the kid uses screen time, the ring drains. A full ring means all budget is available; an empty ring means the daily limit has been reached. The ring circumference is 2 × π × 36 ≈ 226.2.

The one exception is the Blocked state, which uses a segmented ring to show task completion instead of budget.

Animation

The ring should drain with a smooth, continuous linear animation, following the same pattern as the iOS Clock timer ring. Key specs:

  • Timing: linear interpolation on strokeEnd (or equivalent stroke-dasharray value). No easing curves; the drain rate should feel constant and predictable, matching the real passage of time.
  • Update frequency: driven by a 1-second interval timer that updates the remaining time. The ring animates smoothly between each 1-second tick using a 1s linear transition, creating a continuous drain effect.
  • State transitions: when the card switches between states (e.g. from unlock_available to timed_unlock), the ring should animate to its new value with a 0.3s ease-out transition rather than jumping instantly.
  • Reduced motion: when prefers-reduced-motion: reduce is active, skip the smooth drain and update the ring in discrete 1-second steps with no transition.

Interactivity

Only two states respond to taps:

  • Unlock Available: opens the unlock sheet where the kid selects a duration and spends gems.
  • Free Unlock: navigates to the screen time detail view.

All other states are non-interactive. The card does not show a pressed/highlight state for non-tappable variants. Apply .st-card--tappable to add the pointer cursor on interactive variants.

Content

Screen time card copy should:

  • Always use "Screentime" (one word) as the label
  • Keep titles short (2-4 words) so they fit without wrapping at 370px
  • Use MM:SS format for countdown titles (zero-padded minutes and seconds)
  • Frame locked states positively: "Scheduled Break" not "Blocked by schedule", "All Done for Today" not "Limit exceeded"
  • Keep subtitles actionable when possible: "Spend gems for screen time", "Complete tasks to earn more"

Accessibility

  • Semantics: Each card uses role="article" with an aria-label that combines the current state and key value, for example aria-label="Screen time active, 16 minutes 15 seconds remaining".
  • Keyboard: Only tappable variants (Unlock Available, Free Unlock) are in the tab order. Pressing Enter or Space activates the card.
  • Touch target: The entire card surface (370 × ~222px) is the tap target, well above the 44px minimum.
  • Motion: The countdown timer updates every second. When prefers-reduced-motion: reduce is active, the ring progress should update in discrete steps rather than smooth animation.
  • Colour independence: All states are distinguishable by icon and text alone. The card always uses the same green background, so no information is conveyed by colour changes.
