Avatar Row
A horizontal list of labeled Avatars representing family or group members. Each item pairs an avatar with a short name label, making it easy to identify and select people at a glance.
  • ✓ Use to display family members in the village or family view.
  • ✓ Use to show participants in challenges, coaching sessions, or activities.
  • ✓ Keep labels short — first name or role (Mom, Dad, Nana).
  • ✓ Use the scrollable variant when items may exceed the container width.
  • ✗ Don’t use for overlapping/stacked avatars — use Avatar Group instead.
  • ✗ Don’t mix avatar sizes within a single row.

  • Avatar Row — Horizontal flex container with consistent gap between items. Supports scrollable overflow.
  • Avatar Item — A column that stacks an Avatar above a name label, centered and constrained to a max-width.
  • Avatar — Reuses the existing Avatar component (Rive character, silhouette, initial, or Sprout).
  • Name label — Short text (first name or role) centered below the avatar. Truncated with ellipsis when it exceeds the item max-width.
  • Add action (optional) — A dashed-border circle with a + icon at the end of the row. Invites the user to add a new member. Uses --border-secondary dashed border and --text-quaternary icon color.

Accessibility

  • The row container should have role="list" and a descriptive aria-label (e.g., "Family members").
  • Each avatar item should have role="listitem".
  • The visible name label provides the accessible name — no additional aria-label is needed on the avatar when the label is present.
  • For scrollable rows, ensure focus navigation works with keyboard (left/right arrow keys).
  • Rive <canvas> elements should have pointer-events: none to prevent capturing touch events.
