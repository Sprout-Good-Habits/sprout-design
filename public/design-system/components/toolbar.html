Toolbar - Top
The top toolbar provides navigation, context, and actions at the top of every screen. It adapts its layout and content based on where the user is in the app — browsing, viewing detail, chatting, or completing a quest.
  • ✓ Use a toolbar at the top of every screen to provide navigation and contextual actions.
  • ✓ Choose the style that matches the user’s current context — browsing, detail, quest, or chat.
  • ✗ Don’t stack multiple toolbars. Use one toolbar per screen.
  • ✗ Don’t use the Quest toolbar outside of exercise flows — it’s tied to linear progress.

  • Leading — Primary navigation (Back, Cancel/X, Shield). Always leftmost. Accepts 1 Button utility.
  • Center — Title, title + subtitle, or progress bar. Positioned absolutely (Default, Chat) or in flow (Large Title, Compact Large).

Scroll behavior

Large Title uses an animated collapse on scroll, similar to iOS large title navigation bars. As the user scrolls down, the large title row smoothly shrinks and slides up into the compact inline position. Scrolling back to the top reverses the animation.

Note: This behavior applies only to Large Title. Quest map shares the same two-row structure but does not collapse on scroll — the map content below needs full visible space, and collapsing the title row would not leave enough room.

Scroll down

Large title fades out and collapses. Title moves inline between leading/trailing buttons — becoming Compact Large layout.

⬇️
Scroll to top

Inline title fades out. Large title row expands back, restoring the full Large Title layout.

  • Trigger: Scroll offset of the page content. Collapse begins immediately on scroll and completes within ~60px of scroll distance.
  • Expanded (top): Full Large Title layout — two-row with controls above, large title + subtitle below.
  • Collapsed (scrolled): Single-row Compact Large layout — title moves inline next to the leading/trailing buttons, subtitle hidden.
  • Animation: The large title row opacity and height animate to 0 while the inline title fades in. Total transition duration: 300ms ease. Driven by scroll position, not a timed animation — the transition tracks the user’s finger/scroll wheel.
  • Subtitle: Fades out with the large title row. Not shown in the collapsed state.
  • Elastic overscroll: On platforms that support rubber-banding (iOS), the large title can stretch slightly larger when pulling past the top. This is native platform behavior and doesn’t need custom implementation.
  • Quest map excluded: Quest map uses the same two-row layout as Large Title but stays fixed — no collapse on scroll. The map content requires full viewport space.

Kid Home scroll behavior — Alt background

On the Alt background (older kid home), everything scrolls together as a single page. The toolbar uses a frosted glass effect that adapts its tint to the content behind it, transitioning through three states: transparent, frosted sky, and frosted grass.

Main background (younger kid): Grass is pinned at the bottom of the viewport and content scrolls behind it. Because sky is always behind the toolbar, it only uses transparent and frosted sky states — frosted grass never triggers.

Note: The iOS status bar (time, signal, battery) is part of the sticky toolbar group — it scrolls and sticks together with the toolbar buttons.

  • Trigger: Scroll position of the screen content. Transition thresholds are based on the grass edge position relative to the toolbar bottom.
  • Transparent (top): No background, no blur. Default state when the user has not scrolled.
  • Frosted sky: Activates after ~10px of scroll while sky is still behind the toolbar. Background: rgba(224, 242, 254, 0.8) with backdrop-filter: blur(16px).
  • Frosted grass: Activates when the grass edge scrolls past the toolbar bottom. Background: rgba(134, 203, 60, 0.8) with backdrop-filter: blur(16px).
  • Transition: Background color animates over 350ms using cubic-bezier(0.32, 0.72, 0, 1) (iOS spring). Driven by scroll position, not a timed animation.

Accessibility

  • Landmark: Wrap the toolbar in <nav aria-label="Top navigation">.
  • Back button: Use aria-label="Go back".
  • Cancel button: Use aria-label="Close".
  • Icon-only buttons: Every Button utility without visible text needs an aria-label.
  • Progress bar: Use role="progressbar" with aria-valuenow, aria-valuemin="0", aria-valuemax="100".
  • Stat counters: Pair each emoji with aria-label (e.g., aria-label="Fire: 100").
  • Title: Use <h1> or role="heading" aria-level="1" for screen name announcement.
