  • ✓ For recurring daily tasks kids complete on their home screen

  • ✗ Not for one-off actions or navigation. Use List Item or Buttons instead
  • ✗ Not for interactive elements inside the card other than the CTA. Use a Sheet for additional actions

Age tiers

The task card adapts its format based on the kid's age tier. Younger kids get a visual, card-based experience; older kids get a more structured list.

TierFormatLayoutRationalePrototype
Tier 2 (younger kids) Carousel Card Horizontal swipeable carousel, one card fills most of the screen Large, visual cards with big emoji are easier for younger kids to scan and tap Tier 2 prototype
Tier 3 (older kids) Task Row Vertical scrolling list grouped by time of day (Now, Earlier, Up Next) Compact rows show more tasks at once, matching older kids' ability to scan lists Tier 3 prototype
Both tiers Anytime Tile 2-column scrollable grid behind an "Anytime" tab Shared format for optional tasks that can be done at any time See both prototypes above
  • Both tiers share the same tasks, colors, emoji, badge data, and state progression. The only difference is the visual format
  • Anytime tiles use the same 2-column grid layout in both tiers
  • A task's color assignment must be consistent across Tier 2 and Tier 3 (e.g., if "Practice Piano" is violet in Tier 2, it must also be violet in Tier 3)

Tasks are split into two categories, each with its own tab and visual treatment.

  • Required tasks appear under the "Today" tab. These are parent-assigned tasks that must be completed by end of day. They show full state progression (not started, submitted, rejected, claimable, claimed).
  • Anytime tasks appear under the "Anytime" tab. These are optional, self-directed activities a kid can do whenever they want. They use the compact tile format in a 2-column scrollable grid.
  • The tab bar sits at the top of the task area and shows a count badge on the "Today" tab indicating how many required tasks remain.
  • Each task has a fixed color that stays consistent across states. Adjacent required tasks should use different colors for visual variety.

Emoji contrast

TossFace emoji have dominant hues that can blend into same-hue card backgrounds. Always check that the chosen emoji has enough visual contrast against its card color.

  • ✗ Never pair a monochrome emoji with a same-hue card (e.g., 🌱 seedling on green, ⭐ star on yellow)

Content guidelines

  • Titles should start with a verb and stay to 1 to 3 words ("Practice Piano", "Read for 20 min")
  • Descriptions are optional. One short line of supporting context that truncates with an ellipsis
  • Choose a TossFace emoji that directly represents the activity. Avoid abstract or ambiguous symbols. Ensure the emoji has enough contrast against its card color (see Emoji contrast table above)
  • CTA copy should be a short verb phrase, optionally with a leading emoji ("Start", "Take Photo")
  • Do not repeat the same color on adjacent cards in the carousel
  • Each task keeps a fixed color across all states and time-of-day views

Accessibility

  • Touch targets: Carousel cards (278px) and task rows (full width, 52px height) exceed the 44px minimum. Anytime tiles at 136px wide also meet the requirement
  • Contrast: White text on -500 color backgrounds meets WCAG AA for large text. Card badge text uses -700 on -200 for sufficient contrast
  • Carousel navigation: The carousel supports horizontal swipe gestures. Consider adding keyboard arrow key navigation for assistive devices
  • Labels: Each card should have an aria-label combining the task name and state (e.g., "Talk to Sprout, not started, 3 day streak, 10 gems")
  • Status badges: The emoji status badges on task rows convey state visually. Pair with descriptive text (e.g., "Reviewing...") for screen readers
  • Completed state: The overlay dims the card but does not hide content from screen readers. Use aria-disabled="true" on completed cards
