Tier 2 (younger kids)

A large, full-color card with a big emoji, task name, streak and gem badges, and a CTA button. Designed for younger kids who benefit from visual, tap-friendly elements.

Anatomy

  • Background (required) — Full-bleed -500 colour fill, 370×392px with 16px radius
  • Badges (optional) — Streak (left) and gem (right) pill badges at the top, color-matched to the card
  • Title (required) — 14px semibold, white. Optional description below at 10px, 85% opacity
  • Emoji (required) — 96px TossFace emoji centered in the card body
  • CTA (required in active states) — White secondary-style button pinned to the bottom. Disabled in reviewing state, hidden in completed state
  • Completed overlay (claimed state only) — rgba(0,0,0,0.4) overlay covering the full card

Tier 3 (older kids)

A two-part card with a white row on top and an expandable action area below. The row shows the emoji box, task info, and trailing badges. Rejected and Claimable states expand to reveal a CTA button.

Anatomy

  • Card wrapper — --bg-secondary background, --radius-2xl (16px) radius. Contains the row and optional action area
  • Row — White background, --radius-2xl, --spacing-lg (12px) padding, --spacing-md (8px) gap
  • Emoji box (required) — 52×52px with -500 colour fill, --radius-sm (6px), --spacing-md (8px) padding, containing a 36px TossFace emoji
  • Status badge (conditional) — Status badge overlaying the bottom-right of the emoji box: 🔄 reviewing, ❎ rejected, ✅ claimable
  • Task name (required) — --font-size-text-md bold in --text-secondary
  • Time & description (optional) — --font-size-text-xs in --text-quaternary / --text-tertiary
  • Trailing badges — Inline badges for streak and gem counts, aligned right
  • Action area (Rejected & Claimable only) — Expands below the row with a centered description and a 50px CTA button

State behavior

Both tiers share the same five states. The task's color stays fixed; only the CTA, overlay, and status indicators change.

StateTier 2 behaviorTier 3 behavior
Not started CTA reads "Start" Row only, no status badge, no action area
Rejected CTA reads "Try again" ❎ status badge, action area expands with description + "Try again" CTA
Submitted CTA disabled with "Reviewing..." text 🔄 status badge, row only (no action area)
Claimable CTA reads "Claim" with gem emoji ✅ status badge, action area expands with description + "Claim" CTA
Claimed Dark overlay covers card, CTA hidden, streak increments Row only, description reads "Done!", streak increments
  • Streak badge shows a muted flame (filter: grayscale(1) brightness(2)) when streak is 0, full colour when ≥ 1
  • When a task is claimed, the streak badge increments by 1

Badges

Streak and gem badges appear on both formats. Tier 2 uses pill badges color-matched to the card; Tier 3 uses inline badges. See the Badges component for full documentation, variants, color mapping, and token reference.

Content guidelines

  • Titles should start with a verb and stay to 1 to 3 words ("Practice Piano", "Read for 20 min")
  • Descriptions are optional. One short line of supporting context that truncates with an ellipsis
  • Choose a TossFace emoji that directly represents the activity. Avoid abstract or ambiguous symbols
  • CTA copy should be a short verb phrase, optionally with a leading emoji ("Start", "Take Photo")
  • Do not repeat the same color on adjacent cards
  • Each task keeps a fixed color across all states and both tier formats

Accessibility

  • Touch targets: Carousel cards (370px) and task rows (full width, 52px height) both exceed the 44px minimum
  • Contrast: White text on -500 color backgrounds meets WCAG AA for large text. Card badge text uses -700 on -200 for sufficient contrast
  • Labels: Each card should have an aria-label combining the task name and state (e.g., "Talk to Sprout, not started, 3 day streak, 10 gems")
  • Status badges: The emoji status badges on task rows convey state visually. Pair with descriptive text (e.g., "Reviewing...") for screen readers
  • Completed state: The overlay dims the card but does not hide content from screen readers. Use aria-disabled="true" on completed cards
