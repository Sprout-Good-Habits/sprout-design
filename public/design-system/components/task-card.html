Task Card
Colourful cards that represent daily tasks and habits in the kid home screen carousel. Each card shows a task with its emoji, streak, gem reward, and a CTA to begin or continue.
  • ✓ For recurring daily tasks kids complete on their home screen, like “Talk to Sprout” or “Read for 20 min”.
  • ✓ When you need a visual, swipeable overview of everything a kid can do today.
  • ✗ Don’t use task cards for one-off actions or navigation — use List Item or Buttons instead.
  • ✗ Don’t place interactive elements inside the card other than the CTA — use a Sheet for additional actions.

Anatomy

  • Background (required) — Full-bleed -500 colour fill, 278×315px with 24px radius.
  • Badges (optional) — Streak (left) and gem (right) pill badges at the top. -200 background, -700 text.
  • Title (required) — 22px medium weight, white. Optional description below at 12px, 85% opacity.
  • Emoji (required) — 120px TossFace emoji centered in the card body.
  • CTA (required in default state) — White secondary-style button pinned to the bottom. Uses the same shadow pattern as Buttons secondary. Disabled in reviewing state, hidden in completed state.
  • Completed overlay (completed state only) — rgba(0,0,0,0.4) overlay covering the full card.

  • Not started — CTA is active with a generic action label (e.g., “Start”). Streak badge may show a muted flame.
  • Rejected — A parent has rejected the submission. CTA reads “Try again” and the card remains interactive.
  • Submitted — The task has been submitted and awaits parent review. CTA is disabled with “Reviewing…” text.
  • Claimable — A parent has approved the submission. CTA reads “Claim” with a gem emoji so the kid can collect their reward.
  • Claimed — The reward has been collected. A dark overlay covers the card and the CTA is hidden.

Content guidelines

  • Titles should start with a verb and stay to 1–3 words (e.g., “Practice Piano”, “Read for 20 min”)
  • Descriptions are optional — one short line of supporting context that truncates with an ellipsis
  • Choose a TossFace emoji that directly represents the activity — avoid abstract or ambiguous symbols
  • CTA copy should be a short verb phrase, optionally with a leading emoji (e.g., “Let’s Chat”, “Take Photo”)
  • Don’t repeat the same colour on adjacent cards in the carousel

Accessibility

  • Touch targets: The full 278×315px card area is a tap target. The CTA button within meets the 44px minimum touch width.
  • Contrast: White text on -500 color backgrounds meets WCAG AA for large text. Badge text uses -700 on -200 for sufficient contrast.
  • Carousel navigation: The carousel container should support horizontal swipe gestures and keyboard arrow key navigation.
  • Labels: Each card should have an aria-label combining the task name and state (e.g., “Talk to Sprout — not started, 0 day streak, 5 gems”).
  • Completed state: The overlay dims the card but does not hide content from screen readers. Use aria-disabled="true" on completed cards.
