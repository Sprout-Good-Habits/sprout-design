Task Card
Represents a single daily task or habit on the kid home screen. The card has five visual states across its completion lifecycle, and two physical formats depending on the child's age tier. Ground truth: apps/child/components/tasks/TaskCard.tsx and apps/child/components/home/TaskListCard.tsx in the sprout-app repo.
  • ✓ For recurring daily tasks a kid completes on their home screen
  • ✓ When you need a visual overview of what a kid has to do today, with clear state feedback
  • ✗ Not for one-off actions or navigation. Use Tile or Buttons instead
  • ✗ Not for interactive elements inside the card other than the whole-card tap. Use a Sheet for additional actions

State machine

Both formats share the same five visual states. The TypeScript type is TaskCardVisualState = 'available' | 'inProgress' | 'submitted' | 'completed' | 'rejected', defined in apps/child/lib/task-visual.ts. Only available, inProgress, and rejected are tappable. The visual state flows through the shared TaskBadge component (apps/child/components/tasks/TaskBadge.tsx), which flips colour + icon + interactivity + subtitle copy.

State Badge Icon Subtitle Gems line Tappable
available gray-300 Flame (muted) Task description Earn X gems Yes
inProgress gray-300 Loading spinner Try it again Earn X gems Yes
submitted gray-400 (fg.disabled) Hourglass Reviewing... Earn X gems (if reward) No
completed red-400 Flame (full color) + streak number Done! Earned X gems No
rejected yellow-400 Warning triangle Try again Earn X gems Yes
  • available → submitted on child submission
  • submitted → completed when parent approves and canSubmitMore is false
  • submitted → available (displayed as inProgress) when parent approves and canSubmitMore is true and completedToday > 0
  • submitted → rejected when parent rejects
  • rejected → available when child retries
Note on available vs inProgress. Both states share the same task badge colour (gray-300) and flame icon. They are differentiated by the CardAvatar progress ring: available has no ring (or a 0% ring), while inProgress shows a partial ring (driven by completedToday / dailyTarget). The subtitle also changes from the task description to "Try it again". The current sprout-app TaskBadge.tsx uses brand-500 for the inProgress badge — this is a known drift that should be resolved in code to match Figma (which keeps the badge gray and uses the CardAvatar progress ring as the primary visual signal).

Carousel Card (Tier 1)

Full-width white card used by HomeCarousel for ages 4-6. Width is screenWidth - 32 (approximately 360px) and height is a fixed 220px. The task avatar sits in the top-left, the task name + subtitle + gems line in the bottom-left, and the 40 × 112 px bookmark badge overlays the top-right at right: 24.

Anatomy

  • Background (required) — --bg-primary white, --radius-2xl (16px) radius, soft drop shadow
  • Avatar (required) — 96 × 96 px rounded square with the task emoji. Renders via CardAvatar component
  • Task name (required) — 30 px / 32 line-height, --font-weight-bold, --text-primary, up to 2 lines
  • Subtitle (optional) — --font-size-text-sm, --text-tertiary, single line with ellipsis. Content depends on state
  • Gems line (optional) — --font-size-text-sm, --text-tertiary, inline 14 px TossFace gem glyph. Rendered only when the task has a gem reward (gems > 0). Tasks with no reward never show a gems line, in any state
  • Task badge (required) — 40 × 112 px bookmark at top-right. Colour + icon + optional streak number driven by visual state. See Task badge section below

Task List Card (Tier 2 & Tier 3)

Full-width white row used by HomeTaskScroll for ages 7-13. Fixed 80px height with a 52 px avatar on the left, stacked text in the middle, and a smaller 40 × 68 px bookmark badge at the top-right (right: 12). Tier 2 and Tier 3 share the same component, with only font-size, touch-target, and border-radius differing via the useAgeTier() config in apps/child/constants/ageTierConfig.ts.

Anatomy

  • Background (required) — --bg-primary white, tier-specific radius (20px at Tier 2, 16px at Tier 3)
  • Avatar (required) — 52 × 52 px rounded square with the task emoji (Default type) or the Rive village character (Village type). See the Type variants section
  • Task name (required) — --font-weight-bold, --text-primary. Tier 2: 17px. Tier 3: 15px. Single line with ellipsis
  • Subtitle (optional) — --text-quaternary. Tier 2: 13px. Tier 3: 11px. Single line with ellipsis
  • Gems line (optional) — --text-quaternary with inline 12 px TossFace gem glyph. Rendered only when the task has a gem reward (gems > 0). Tasks with no reward never show a gems line, in any state
  • Task badge (required) — 40 × 68 px bookmark at top-right with 16 px icon
  • Action area (Tier 3 rejected only) — The card expands to ~186 px tall and reveals an action area below the row, with --bg-secondary background, a centered description, and a full-width CTA button. The CTA is the shared Button component (.btn-sprout.btn-secondary.btn-md). Tier 2 stays at 222 px in all states

Task badge

The 40 px bookmark tab that hangs off the top edge of every task card. It's the single source of truth for visual state and is shared across both card formats. Source: apps/child/components/tasks/TaskBadge.tsx.

  • Width — 40 px (constant)
  • Height — 112 px on Carousel Card, 68 px on Task List Card
  • Icon size — 24 px when badge height is ≥ 60, 16 px otherwise
  • Tail — 8 px diagonal curl at the bottom-right, rendered as a single SVG <path>
  • Colour — Full-bleed fill of the state colour (gray-300, brand-500, fg.disabled, red-400, yellow-500)
  • Streak overlay — Only in completed state when streak > 0. White --font-size-text-xs bold number below the flame icon

Type variants

Each task card has two Type variants that change how the avatar slot renders. The card chrome, state machine, and task badge are identical between the two — only the avatar content differs.

Default Task emoji in a coloured rounded square General tasks where the activity itself is the visual — reading, homework, chores, piano
Village The kid's personalized Rive village character (character2.91.riv) Tasks where the kid's identity is more relevant than the activity — onboarding, goal milestones, identity-driven prompts

CardAvatar

The CardAvatar is the avatar rendered inside a task card. It has two content variants (Default and Village) and an optional progress ring that shows streak progress toward a goal — used to visualize how close the kid is to a reward, level, or village-character unlock. Eight discrete ring states cover 0 → 87.5%, looping back to 0 at 100%. Source: apps/child/components/common/CardAvatar.tsx in sprout-app. Previously labelled Goal icon / GoalIcon in Figma.

Anatomy

  • Circle container — Filled circle in --brand-100, border-radius: var(--radius-full), overflow: hidden so child content (emoji or Rive canvas) is clipped to the circle
  • Avatar content — Either the task emoji (Default variant, TossFace glyph sized 44 px on big / 26 px on small) or the Rive village character (Village variant, character2.91.riv loaded as a canvas inside the circle and clipped to the head-and-shoulders frame via a negative top offset)
  • Track ring — Full-circle stroke in --gray-300, 4 px wide on big, 3 px on small. Always visible behind the progress arc when a ring is present
  • Progress arc — Same radius as the track, stroked in --brand-500, rounded caps, rotated so 0% starts at 12 o'clock. Uses stroke-dasharray with the filled length equal to circumference × progress
  • Layering — The progress ring is rendered as an SVG overlay on top of the circle container, so the same ring works identically on both Default and Village variants

Content guidelines

  • Titles should start with a verb and stay to 1 to 3 words ("Practice Piano", "Read for 20 min")
  • Subtitles are state-driven, not author-driven. They come from getTaskSubtitle() in task-visual.ts and fall back to the task description only in the available state
  • Choose a TossFace emoji that directly represents the activity. Avoid abstract or ambiguous symbols
  • Gems copy is tense-aware: "Earn X gems" in every state except completed, which reads "Earned X gems"
  • The gems line is fully optional. It is rendered only when the task has a gem reward (gems > 0 in getTaskGemsText()). Tasks with no gem value never show a gems line in any state

Accessibility

  • Touch targets — Both formats exceed the 44 px minimum. Tier 1: full 360 × 220. Tier 2/3: full card row at 80 px
  • Interactive states — Only available, inProgress, and rejected wire up onPress. In submitted and completed, the card renders with disabled={true} and aria-disabled="true"
  • Label — Each card should have an aria-label combining the task name and state, e.g., "Talk to Sprout, available, earn 10 gems"
  • Badge semantics — The task badge is decorative and colour-coded. Always pair with the subtitle text so the state is conveyed to screen readers
  • Contrast — White text / icons on the brand, red, yellow, and disabled-gray badge backgrounds meet WCAG AA at 16 px and above
