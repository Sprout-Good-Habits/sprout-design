Toast
Toasts provide brief, non-blocking feedback about an action or event. They appear at the bottom of the screen and auto-dismiss after a short delay.
  • ✓ For confirming completed actions: saved, sent, deleted, unlocked.
  • ✓ For surfacing non-critical errors that don’t block the flow.
  • ✓ For tips, hints, or brief system status updates.
  • ✗ Don’t use for persistent messages — use an action prompt instead.
  • ✗ Don’t use for actions that require a decision — use a sheet or dialog.
  • ✗ Don’t use for critical errors that block the user — use a banner with role="alert".

Anatomy

A toast is a floating card with an icon, text content, an optional close button, and a progress bar that indicates auto-dismiss timing.

  • Icon (optional) — 20px Toss Face emoji in a 24px wrapper. Reinforces the message at a glance.
  • Label (required) — text-md bold (16px/24px). Short, descriptive headline.
  • Description (optional) — text-sm regular (14px/20px). Additional context below the label.
  • Action (optional) — text-sm bold (14px/20px), --brand-700. Provides a direct path to follow up.
  • Close button (optional) — 24px icon. Allows manual dismissal.
  • Progress bar — 4px bar at the bottom edge. Fills from left to right to show time remaining before auto-dismiss.

Placement

Toasts appear at the bottom center of the screen, above the safe area. They float over content and should not push the layout.

Bottom center (default)
Screen content
✅
Saved successfully
  • ✓ Position at the bottom center, 24px above the safe area.
  • ✓ Only show one toast at a time — queue additional toasts.
  • ✗ Don’t place toasts at the top of the screen — that’s reserved for banners.
  • ✗ Don’t cover navigation or critical interactive elements.

Dismissing a toast

Toasts can be dismissed in three ways:

  • Auto-dismiss — the progress bar fills over 3–5 seconds, then the toast slides away. Use 5s when an action link is present.
  • Close button — tapping the X icon removes the toast immediately.
  • Swipe — swiping the toast down or sideways dismisses it (native gesture).

Pause the auto-dismiss timer when the user is hovering over or has focus inside the toast.

Animation preview

Content guidelines

  • ✓ Keep the label short — lead with the outcome, not the process (“Saved” not “Your changes have been saved”).
  • ✓ Use the description to add context only when the label isn’t enough.
  • ✓ Choose an icon that matches the message content, not just the severity level.
  • ✗ Don’t use technical jargon — keep language simple for parents and kids.
  • ✗ Don’t use the action link for navigation — link to the relevant follow-up, not another page.

Accessibility

  • Role: Use role="status" with aria-live="polite" for success, warning, and info toasts. Use role="alert" for error toasts so screen readers announce them immediately.
  • Close button: Must have aria-label="Dismiss" and be keyboard accessible.
  • Action link: Use a semantic <button> element, not a styled <div>.
  • Auto-dismiss: Pause the timer on hover and focus so users have time to read and interact.
  • Focus management: When a toast is dismissed, do not steal focus from the user’s current position.
  • Motion: If animating entry/exit, respect prefers-reduced-motion — use opacity fade instead of slide.
