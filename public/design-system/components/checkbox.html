Checkbox
Checkboxes let users select one or more items from a list, or toggle a single option that requires explicit confirmation before taking effect.
  • ✓ For multi-select lists where the user can pick more than one option.
  • ✓ For consent or agreement confirmations (e.g. "I agree to the terms").
  • ✗ Don't use a checkbox for mutually exclusive options — use radio buttons instead.
  • ✗ Don't use a checkbox for instant on/off toggles — use a switch instead.

Anatomy

  • Box (required) — 24×24px with 6px radius. Unchecked: white with --border-secondary. Checked: --brand-500 fill.
  • Check icon (checked only) — White checkmark SVG centered in the box.
  • Label (required) — Regular text-md in --text-primary. Placed to the right of the box.
  • Description (optional) — Regular text-sm in --text-secondary. Below the label.

Content guidelines

  • Labels should clearly describe the option being selected
  • Use positive framing — describe what will happen when checked, not what won't happen
  • Keep labels concise — one line when possible
  • In a group, each label should make sense independently

Accessibility

  • Role: Use native <input type="checkbox"> or role="checkbox" with aria-checked="true|false".
  • Labels: Associate a visible label via <label for> or aria-labelledby.
  • Keyboard: Toggle with Space. Must be focusable with Tab.
  • Groups: Wrap related checkboxes in a fieldset with a legend, or use role="group" with aria-labelledby.
  • Contrast: The brand-500 checked fill and white check icon meet WCAG AA. The unchecked border (--border-secondary) on white meets 3:1 for UI elements.
  • Touch targets: The 24px box is below the 44px minimum — ensure the tap area extends to include the label row (full 370px width).
