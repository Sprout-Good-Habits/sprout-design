Avatar
The avatar is a person’s face in the product. Every village member gets a character that represents them everywhere — profiles, challenges, chat, coaching sessions, family view. The avatar shows how you relate to someone, not just who they are. Users personalize their avatar by choosing a hairstyle, skin tone, clothing, and background color in the character customizer.
  • ✓ Use the Avatar variant whenever the user has customized their village character (Rive).
  • ✓ Use the Silhouette as the default placeholder before a user customizes their character.
  • ✓ Fall back to User initial when neither character nor silhouette data is available.
  • ✓ Use the Sprout variant exclusively for the Sprout mascot character (Rive).
  • ✓ Always let the user’s chosen background color show behind the character.
  • ✗ Don’t use avatars as buttons — wrap in a Button Utility for interactivity.
  • ✗ Don’t hard-code background colors — always respect the user’s saved preference.

When to use

Avatars identify people and characters across the app. Choose the right variant based on available data.

  • Avatar — The primary variant. Renders the user’s village character via Rive with their chosen hairstyle, skin tone, clothing, and background color.
  • Silhouette — Default placeholder before a user has customized their character. Shows a gray human figure SVG on the --bg-sky background.
  • User initial — Fallback for when neither the character nor silhouette is available (e.g., loading states). Shows the first letter of the user’s name.
  • Sprout — Reserved for the Sprout mascot. Uses the same Rive asset with the default Sprout appearance.

When not to use

  • Don’t use Avatar for decorative illustrations — it represents a specific person or character.
  • Don’t add tap behavior to the avatar itself. Wrap it in a Button Utility if it needs to be interactive.

Anatomy

Circle container
Background color (user-selected)
Content: Rive character, silhouette SVG, or initial
2px white border (when inside a group)
  • Circle container — Fully rounded (border-radius: 9999px), clips all content. Sized to one of the five scale steps.
  • Background color — Defaults to --bg-sky (#e0f2fe). Users choose from 10 pastel options in the character customizer.
  • Rive character (Avatar & Sprout) — The village character asset rendered on a <canvas>. Uses the Village-character artboard with State Machine 1.
  • Always grounded — The bottom edge of the container always clips the character. It is never floating in empty space.
  • Silhouette SVG — A static gray figure (--gray-300 / #d5d7da) rendered as an <img>, also anchored bottom-center within the circle.
  • User initial — A single character centered in the circle, using --text-brand-secondary (#026aa2).
  • Border — A 2px white border is added via .avatar-bordered when avatars overlap inside an Avatar Group.

Usage context

Avatars appear in several contexts across the app:

  • Avatar Group — Multiple avatars stacked with −8px overlap inside an Avatar Group. Use .avatar-bordered for the 2px white stroke.
  • Chat toolbar — SM-sized avatars inside a Button Utility in the toolbar trailing slot.
  • Profile / settings — LG or XL size for profile headers and the character customizer.
  • Inline references — XS or SM sizes for compact participant lists or mentions.

Accessibility

  • All avatar variants must have aria-label describing who the avatar represents (e.g., aria-label="Mom's avatar").
  • The Sprout avatar should carry role="img" and aria-label="Sprout".
  • Silhouette avatars should have aria-label with the user’s name even though the image is generic.
  • User initial avatars are self-describing via the visible letter, but should still carry an aria-label with the full name for screen readers.
  • Avatars are decorative when inside an interactive container (Button Utility) — the container provides the accessible label.
  • Rive <canvas> elements should have pointer-events: none to prevent capturing touch events from the container.
