Tile
Tiles are bordered, tappable cards that let users choose between options. Each tile is visually distinct with its own border and shadow, making it ideal for selection flows, exercises, and onboarding.
  • ✓ Use a tile when you want the user to pick from a set of distinct, card-like options.

Leading element

The leading element is an optional 40px .Leading subcomponent with three types. It can be shown or hidden via showLeading.

The leading area is a horizontal container with gap: 4px, so you can stack multiple leading elements side by side. For example, the Sortable variant uses a number badge + emoji, or a number badge + avatar.

Usage guidelines

Types

  • ✓ Use Navigation when making a choice navigates to another screen or step in a flow.
  • ✓ Use Radio to select a single item from a mutually exclusive list.
  • ✓ Use Checkbox to select multiple items from a list.
  • ✓ Use None when no trailing indicator is needed — avatar and label only.
  • ✓ Use Image to select from photos or illustrations — 112px square tiles with an image fill.
  • ✓ Use Sortable for drag-to-reorder exercises — shows a drag handle and supports multiple leading elements (e.g. number badge + emoji).
  • ✓ Use section headers to separate lists with different types on the same page.
  • ✗ Don’t mix different types in the same list.
  • ✗ Don’t use Navigation when the user needs to choose options within a form — use Radio or Checkbox instead.
  • ✗ Don’t use a tile when there is just one primary option — use a button instead.
  • ✗ Don’t use Radio or Checkbox directly in a form, unless used as a full screen.

Leading element

The leading element is an optional 40px .Leading subcomponent (showLeading) with three types: Avatar, Icon, and Swatch.

  • ✓ Use an Avatar when the tile represents a person — family members, leaderboard entries, participants.
  • ✓ Use an Icon (Toss Face emoji) for activity or category lists.
  • ✓ Use a Swatch for color selection — a 40px rounded square with a color fill.
  • ✓ Always respect the user’s profile background color on the avatar.
  • ✗ Don’t mix different leading types in the same list.
  • ✗ Don’t hard-code avatar background colors — always use the color from the user’s profile.

Anatomy

Checkbox, Radio, Navigation, and None tiles are composed of up to four parts. All parts are configurable via boolean props: showLeading, showLabel, showDescription. The Image type is a compact 112px square containing only an image.

Validation states

Validation states are applied after the user submits an answer in exercises and quizzes. They are non-interactive (pointer-events: none) — the user can’t change their selection after validation.

  • ● .success — The option the kid selected and it was correct.
  • ● .error — The option the kid selected but it was wrong.
  • ● .warning — The correct answer the kid didn’t pick. Flashes yellow for ~1.5s then returns to default so the kid can try again. Use .warning-static in docs/demos to show the yellow state without animation.
  • ● .disabled — Options that were not selected and are not the correct answer. Muted label, no shadow, non-interactive.

Content guidelines

Tile copy should:

  • Keep labels short — ideally 2–4 words
  • Start with a verb or noun that names the option
  • Use sentence case (only capitalise the first letter of the first word)
  • Use the description for extra context only when the label isn’t self-explanatory
  • Avoid ending labels or descriptions with a period

Accessibility

Tiles are interactive selection controls. They must be usable by all input methods.

  • Role: Checkbox type → use role="option" in a role="listbox", or native <input type="checkbox">. Radio type → use role="radio" in a role="radiogroup", or native <input type="radio">. Navigation type → use <a> or role="link".
  • Keyboard: Items must be selectable with Enter or Space. Arrow keys should move focus between items.
  • Focus ring: A visible 2px focus ring appears on keyboard navigation. Do not remove :focus-visible outlines.
  • ARIA state: Use aria-selected="true" for checked items and aria-selected="false" for unchecked. Radio groups should use aria-checked.
  • Contrast: Labels (--text-primary on --bg-primary) and descriptions (--text-secondary on --bg-primary) both pass WCAG AA.
  • Touch targets: The entire row (minimum 56px tall) is the tap target, exceeding the 44×44px guideline.
