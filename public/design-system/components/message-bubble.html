Message Bubble
Message bubbles display individual chat messages in a conversation. Incoming messages appear left-aligned with an optional avatar, while outgoing messages are right-aligned with a distinct blue background.
  • ✓ Use for displaying text, voice, and media messages in chat conversations.
  • ✓ Pair with the Message Composer for a complete chat experience.
  • ✗ Don't use for system notifications or alerts. Use Toast or Feedback Banner instead.
  • ✗ Don't nest interactive components inside bubbles.

With sender name

Typing

Anatomy

  • Avatar (optional, incoming only) — 32px circle with sky-blue background. Uses the Sprout character positioned inside the circle (head cropped) or participant photo.
  • Sender name (optional, incoming only) — text-xs (12px/16px) label in --text-secondary. Appears above the bubble in group conversations.
  • Bubble (required) — Rounded container with 24px radius. White for incoming, --brand-200 blue for outgoing.
  • Text content (required for text type) — text-sm (14px/20px) in --text-primary. Wraps at max-width 256px.
  • Media grid (optional) — 104px square images, 20px radius, 4px gap. Wraps above the text bubble.
  • Waveform row (voice type) — Play/pause button (gray circle, toggles between play and pause icon) + waveform bars inside the bubble.
  • Transcription label (voice type) — text-xs (12px) in --text-secondary. "Transcribing..." (regular) or "Show transcript" (bold).
  • Deleted indicator (deleted type) — x-circle icon (20px) + "Message was deleted" in --text-secondary.

Content guidelines

  • Message text should be displayed exactly as typed by the user
  • The "Message was deleted" label is a system string, not user-editable
  • "Transcribing..." and "Show transcript" are fixed labels for voice message states
  • Sender names should be the participant's display name, not a username or ID
  • Media grids wrap naturally. Up to 5 images per message at phone width

Accessibility

  • Role: The message list container should use role="log" with aria-live="polite" so new messages are announced by screen readers.
  • Message structure: Each message should be a distinct element with role="listitem" or within a role="list" container.
  • Direction: Indicate sent vs received using aria-label (e.g. "Mali said: Hello" or "You said: Hello").
  • Voice messages: The play button needs aria-label="Play voice message". Transcription status should be announced.
  • Deleted messages: Screen readers should announce "Message was deleted" rather than leaving a silent gap.
  • Media: Image attachments need descriptive alt text or aria-label="Photo attachment".
  • Typing indicator: Use aria-label="Typing" and role="status" so it's announced once.
