Progress Bar
Linear progress indicator used inside the Quest toolbar. Shows how far through an exercise set the kid has progressed. Supports animated fill transitions, streak messages, and spark effects on correct answers.
  • ✓ Use inside the Quest toolbar to show exercise completion progress.
  • ✓ Use when the total number of steps is known and progress is linear.
  • ✗ Don’t use for indeterminate loading — use the Loading spinner instead.
  • ✗ Don’t use outside of exercise/quest flows. Other progress indicators (e.g., profile completion) should use a different pattern.

Anatomy

The progress bar is composed of four elements, all contained within the Quest toolbar context.

Message (optional) 3 in a row!
Inner highlight
Track Fill (progression)
  • Track (required) — 16px tall pill. Background: --bg-secondary. Full width of the toolbar center slot.
  • Fill / Progression (required) — Animated fill inside the track. Background: --brand-500. Min-width: 32px. Padding: 4px vertical, 8px horizontal.
  • Inner highlight (required) — 4px tall white 50% opacity bar inside the fill, creating a glossy/3D effect.
  • Message (optional) — Bold text-xs label above the bar in --brand-500 (#0ba5ec). Used for streak callouts like “3 in a row!”

Animation: Fill transition

When progress advances, the fill width animates smoothly with cubic-bezier(0.22, 1, 0.36, 1) over 400ms. The animation is triggered by updating the fill’s width value.

Step 1 / 10
  • Duration: 400ms
  • Easing: cubic-bezier(0.22, 1, 0.36, 1) — fast snap followed by smooth deceleration (spring-like feel)
  • Trigger: On correct answer / exercise step completion
  • Direction: Always left-to-right. Progress never decreases during a quest session.

Animation: Spark effect

A diamond-shaped confetti burst appears at the leading edge of the progress fill each time it advances. Tiny sparkle particles — rotated squares matching the Figma “Sparkle” shape — fly outward like celebration confetti. This micro-animation provides pure delight on correct answers.

  • Shape: Diamond-shaped confetti particles — 5–8px rounded squares rotated 45°, matching the Figma “Sparkle” shape (1px border-radius).
  • Count: 5 diamond particles per burst.
  • Position: Particles appear scattered around the right edge of the progress fill. The overlay sits on .progress-bar (not inside the track) to avoid overflow:hidden clipping.
  • Behavior: Each particle pops into place at a random offset around the fill edge, holds briefly, then fades out. No outward flight — they appear in position like a momentary sparkle.
  • Colors: Particles randomly pick from --brand-500, --brand-200, --brand-300, and white.
  • Duration: 400–600ms with 30ms stagger between particles.
  • Easing: cubic-bezier(0.22, 1, 0.36, 1) — same spring-snap feel as the fill.
  • Timing: Spark fires at the end of the fill animation (after the 400ms width transition completes).

Animation: Streak message

Streak messages appear above the progress bar at milestone moments, centered over the fill when possible. They fade in, hold, then fade out.

  • Fade in: 300ms ease — opacity 0 → 1, max-height 0 → 20px
  • Hold: ~2 seconds visible
  • Fade out: 300ms ease — opacity 1 → 0, max-height 20px → 0
  • Text style: Uppercase, bold, --brand-500. Centered over the fill (see positioning rules above).
  • Trigger conditions:
    • 3 in a row — “3 IN A ROW”
    • 5 in a row — “5 IN A ROW”
    • 10 in a row — “10 IN A ROW”
  • Pairing: Always accompanied by the spark effect.

Accessibility

  • Role: Use role="progressbar" on the track element.
  • Values: Set aria-valuenow to the current percentage, aria-valuemin="0", aria-valuemax="100".
  • Label: Add aria-label="Exercise progress" to describe the bar’s purpose.
  • Updates: Use aria-live="polite" on the message element so screen readers announce streak milestones.
  • Motion: Respect prefers-reduced-motion — disable spark particles and use instant width changes instead of animated transitions.
