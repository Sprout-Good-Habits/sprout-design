Conversational Feedback
Pairs a character with a text message to create a conversational moment. The character can speak to the user with full presence using an Avatar Portrait, or deliver compact feedback using a circular Avatar. The message is always a Subtitles component.
  • ✓ Use Conversational when the character is “speaking” — gives the character presence via Avatar Portrait.
  • ✓ Use Feedback for compact encouragement with a leading icon — uses the circular Avatar.
  • ✓ Use Incoming direction when the character initiates (avatar on the leading side).
  • ✓ Use Outgoing direction when the message follows the user’s action (avatar on the trailing side).
  • ✗ Don’t use for user-to-user chat — use Message Bubble instead.
  • ✗ Don’t add buttons or interactive elements inside this component.

When to use

  • Streak celebrations — The character congratulates the user on maintaining a streak.
  • Coaching nudges — A gentle prompt from the character to encourage the user to continue.
  • Task completion — Positive reinforcement after finishing an exercise or quest.
  • Welcome back — A personalized greeting when the user returns to the app.

When not to use

  • Don’t use for real-time chat between users — use Message Bubble.
  • Don’t use for system-level alerts — use Toast.
  • Don’t use for long-form content — the Subtitles text is capped and rotates.

Types

Conversational

The character “speaks” to the user. Uses Avatar Portrait (120 × 176px) for full character presence, paired with centered Subtitles.

  • Container — Flex row, fills parent width, gap: 8px (--spacing-md), items vertically centered.
  • Avatar slot — Either an Avatar Portrait (Conversational) or circular Avatar XL (Feedback). Position determined by direction: leading for Incoming, trailing for Outgoing.
  • Subtitles — Fills the remaining space (flex: 1). Layout variant is determined by type: Centered for Conversational, Left aligned + icon for Feedback.

Accessibility

  • Use role="status" on the container so screen readers announce the feedback without interrupting.
  • Add aria-live="polite" when the component appears dynamically.
  • The avatar is decorative in this context — the Subtitles text carries the meaning. Mark the avatar with aria-hidden="true" or use a descriptive aria-label on the container instead.
