Feedback Banner
A bottom-anchored banner that delivers contextual feedback after an action — quiz results, task completions, warnings, or errors — with a prominent CTA to move forward.
  • ✓ Use a feedback banner when the user needs immediate, full-width feedback tied to an action they just took.
  • ✓ Use to present a clear next step (continue, retry, dismiss) after the outcome is shown.
  • ✗ Don’t use for passive notifications or alerts that don’t require a user action — use Toast instead.
  • ✗ Don’t use for prompts that need additional context or choices — use Action Prompt or Sheet instead.

CTA Options

The feedback banner supports two CTA configurations. Choose the one that best fits the user’s decision context.

Primary only

Use when there is a single clear next step. The full-width primary button makes the path forward obvious.

  • ✓ Use primary only when the action is unambiguous — “Continue”, “Got it”, “Try again”.
  • ✗ Don’t use primary only when the user needs to choose between two meaningful options.

Primary + Secondary

Use when the user has two distinct paths. The secondary button sits left, the primary right. A top border separates the banner from content above.

  • ✓ Place the secondary action (dismiss, skip, go back) on the left and primary (continue, confirm) on the right.
  • ✗ Don’t reverse the button order — primary always goes on the right.

Anatomy

  • Icon (optional) — 32×32px TossFace emoji. Reinforces the feedback sentiment at a glance.
  • Title (required) — Display-xs (24px) bold heading. Communicates the outcome immediately.
  • Subtitle (optional) — Text-md (16px) bold. Adds a secondary label like “Correct answer:”.
  • Description (optional) — Text-md (16px) regular. Provides additional context or explanation.
  • Primary button (required) — Full-width CTA with hierarchy-matched color and 4px raised shadow.
  • Secondary button (optional) — White button with border, paired left of primary in a row.

Placement

The feedback banner is always pinned to the bottom of the screen, overlaying content below. It should appear after the user completes an action like answering a question or submitting a form.

  • ✓ Pin the banner to the bottom of the viewport so it’s always visible.
  • ✓ Animate the banner sliding up from the bottom on appearance.
  • ✗ Don’t place the banner inline within scrolling content — it must anchor to the bottom.

Content

Feedback banner copy should:

  • Lead with the outcome in the title — “Great job!”, “Incorrect”, “Almost there”
  • Be encouraging and supportive, especially for error states — kids should feel motivated to try again
  • Use the subtitle for structured labels like “Correct answer:”
  • Keep descriptions short — one line is ideal, two lines maximum
  • Use clear, actionable button labels — “Continue”, “Got it”, “Try again”
  • Avoid jargon or blame language in error states

Accessibility

  • Live region: The feedback banner should use aria-live="assertive" so screen readers announce the outcome immediately when it appears.
  • Role: Wrap the banner in role="region" with an aria-label describing the feedback type, e.g. “Quiz result”.
  • Focus management: When the banner appears, move focus to the title or primary button so keyboard users can act immediately.
  • Keyboard: All buttons are focusable and activated with Enter or Space. Tab order should go: secondary button (if present), then primary button.
  • Contrast: All hierarchy variants use dark text on light tinted backgrounds, exceeding WCAG AA contrast ratios. Button text (white on solid fills) passes AA for large text.
  • Touch targets: Buttons meet the 44×44px minimum touch target guideline.
  • Motion: If using a slide-up animation, respect prefers-reduced-motion by disabling the transition.
