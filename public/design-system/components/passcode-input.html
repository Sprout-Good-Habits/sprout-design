Input Passcode
Passcode inputs let users enter a 4-digit PIN using individual digit cells. Each cell accepts one character and auto-advances focus to the next. Used for child auth and parent area access.
  • ✓ For child authentication PINs where a full keyboard isn’t needed.
  • ✓ For parent area access codes that add a layer of child-proof security.
  • ✗ Don’t use for passwords or long codes — use a standard text input instead.
  • ✗ Don’t use for non-numeric entry — passcode cells are digit-only.

Anatomy

  • Label (optional) — Bold text-md in --text-primary. Placed above the row.
  • Cell (required) — 4 equal-width cells, 56px tall, 16px radius. White background with --border-secondary border.
  • Dot (filled only) — A centered bullet character (•) in --text-primary. Masks the digit for privacy.
  • Error message (error only) — Regular text-sm in --text-error-primary. Appears below the row.

Content guidelines

  • Labels should describe what the code is for, like “Enter your PIN” or “Parent passcode”
  • Error messages should be short and helpful — “Invalid” or “Incorrect PIN, try again”
  • Don’t reveal the digits — always show dots for entered values

Accessibility

  • Role: Each cell should be an <input type="text" inputmode="numeric"> with maxlength="1" and pattern="[0-9]".
  • Labels: Wrap the group in a fieldset with a legend, or use aria-label on the container. Each cell should have aria-label="Digit 1" etc.
  • Keyboard: Typing a digit should auto-advance focus to the next cell. Backspace should clear and move back. Arrow keys should navigate between cells.
  • Auto-submit: After all 4 digits are entered, the form can auto-submit. Announce the result via aria-live="polite".
  • Error: Associate the error message with the input group using aria-describedby. Use aria-invalid="true" on cells in error state.
  • Touch targets: Each cell is 56px tall and flexible width, exceeding the 44px minimum.
