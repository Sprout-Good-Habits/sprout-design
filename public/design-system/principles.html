Principles
Six non-negotiable rules that keep every implementation faithful to the Sprout design system. Break one and the system drifts.

The Rules

Every page, component, and prototype must follow these six principles without exception.

1

Never hardcode colors

Use var(--token-name), not #hex. If you catch yourself writing a hex value, find the matching token in tokens.css.

2

Never redeclare tokens

Link to tokens.css via <link> — do NOT copy token definitions into :root or <style> blocks. Redeclaring creates drift.

3

Never inline shared assets

If an SVG exists in /brand/assets/, reference it with <img src="...">. Never paste its raw markup into the HTML.

4

Never invent components

Check the component library first. If it exists, use the documented pattern exactly. If it doesn't, propose it before building.

5

Always pair font-size with line-height

--font-size-text-md goes with --line-height-text-md. Using one without the other breaks vertical rhythm.

6

Always use the spacing scale

No magic numbers like padding: 15px. Use var(--spacing-xl) (16px) or the nearest scale step.

Rule 1: Never Hardcode Colors

Every color in the system has a semantic token. Use the token so the palette can evolve without touching every file.

Do color: var(--text-primary); background: var(--bg-secondary); border: 1px solid var(--border-primary);
Don't color: #181d27; background: #fafafa; border: 1px solid #d5d7da;

Rule 2: Never Redeclare Tokens

Link the token stylesheet once. Never copy variable definitions into your own :root or <style> block.

Do <link rel="stylesheet" href="/tokens.css"> /* Then use tokens freely */ .card { background: var(--bg-secondary); }
Don't :root { --bg-secondary: #fafafa; --text-primary: #181d27; } .card { background: var(--bg-secondary); }

Rule 3: Never Inline Shared Assets

SVGs and images in /brand/assets/ must be referenced by path. Inlining bloats the page and breaks sync.

Do <img src="/brand/assets/grass.svg" alt=""> <img src="/brand/assets/character-sprout-avatar.svg" alt="Sprout">
Don't <svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 400 80"> <path d="M0,40 C50,0 100,80..." fill="#86cb3c"/> </svg>

Rule 5: Always Pair Font-Size with Line-Height

Every font-size token has a matching line-height token. Using one without the other breaks vertical rhythm and readability.

Do font-size: var(--font-size-text-md); line-height: var(--line-height-text-md); font-size: var(--font-size-display-sm); line-height: var(--line-height-display-sm);
Don't font-size: var(--font-size-text-md); /* no line-height */ font-size: 16px; line-height: 1.5;

Rule 6: Always Use the Spacing Scale

The spacing scale covers every value you need. No magic numbers, no pixel guessing.

Do padding: var(--spacing-xl); /* 16px */ gap: var(--spacing-lg); /* 12px */ margin-bottom: var(--spacing-4xl); /* 32px */
Don't padding: 15px; gap: 11px; margin-bottom: 30px;

CSS Methodology

How we write and organize CSS across the design system.

Inline Style Blocks

Each page uses a <style> block in the <head> for page-specific CSS. No external component stylesheets — everything is scoped and self-contained.

Shared CSS Conflicts

Global files like shared.css define rules for common class names (e.g., .progress-bar). Always check for conflicts and override explicitly in your page styles.

Inter Font Family

All text uses Inter via var(--font-family-body) or var(--font-family-display). Load from Google Fonts or the system font stack fallback.

TossFace Icons

All emoji icons use the TossFace webfont — never system emoji. Load via @font-face from static.toss.im and set font-family: 'TossFace'.

Additional Guidelines

Beyond the six rules, keep these principles in mind for every implementation.

  • • Semantic tokens first. Use --text-primary, --bg-secondary, --border-primary before reaching for raw scale values like --gray-700.
  • • Raw scales only when needed. Use --brand-300 or --sprout-50 only when no semantic token covers the use case (e.g., decorative accents, custom illustrations).
  • • Verify tokens with Figma. Run get_variable_defs against the Figma file to confirm exact token names — never guess from screenshots.
  • • Accessibility by default. Semantic HTML, proper heading hierarchy, keyboard access, WCAG AA contrast (4.5:1 text, 3:1 UI).
  • • Reference assets by path. All SVGs and images in /brand/assets/ must be loaded via <img src>, never inlined or duplicated.
  • • Animation timing from the scale. Fast feedback at 0.15s, transitions at 0.2s, enters at 300ms ease-out, exits at 250ms ease-in.
Tip: When in doubt, check the existing component pages in the UX library. They demonstrate every pattern and token pairing in context.
