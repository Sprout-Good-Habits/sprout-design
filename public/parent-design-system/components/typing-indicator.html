Typing Indicator

Shows that Sprout is composing a response. Single-row layout with avatar, sender name, "Typing..." label, and optional "Answer now" button. Includes an optional Live Action Stream that surfaces real-time tool calls and processing steps.

Overview

The typing indicator appears while Sprout or a co-parent is composing a response. It uses a compact single-row layout (avatar + name + subline + CTA). For Sprout, the subline is a Live Action Stream that cycles through real-time steps ("Thinking...", "Checking family preferences...", "Typing...") with iOS-style slide transitions. When an action fails, the stream shows the failure and retry in real time. For co-parent, the subline is a static "Typing..." with no action stream and no "Answer now" CTA. When the response is ready, the indicator transitions to a full message with a crossfade animation.

Anatomy

  • .msg-group.msg-incoming.typing-state . Horizontal row layout
  • .msg-avatar.typing-avatar . Avatar as direct child for row alignment
  • .sender-indicator.typing-sender . Name label
  • .sender-indicator-typing - Subline text (12px, text-secondary). For Sprout, cycles through action steps with iOS-style slide transitions. For co-parent, static "Typing..."
  • .typing-cta - "Answer now" button (14px, medium weight, Sprout only)
  • .msg-row.msg-incoming - Hidden row (display:none), populated with avatar + bubble on delivery

Display the typing indicator whenever Sprout begins generating a response. Remove it and replace with the full message once the response is ready.

Live Action Stream

The Live Action Stream is not a separate component. It is the Sprout typing indicator's subline (.sender-indicator-typing) cycling through real-time action steps as Sprout processes a request. Each step slides in with an iOS-style transition (slide up out, slide up in from below). This is Sprout-only; co-parent typing indicators always show static "Typing...".

The typical sequence is:

  • Thinking... - initial state while Sprout begins processing
  • [Action]... - tool-specific steps (e.g. "Checking family preferences...", "Searching habit catalog...", "Opening Google Calendar...")
  • Typing... - final step as Sprout composes the response

Each transition uses cubic-bezier(.32,.72,0,1) for a natural spring-like feel. Steps cycle every ~900ms. On delivery, the action stream transitions smoothly to the reasoning indicator ("Show thinking").

Error recovery (retry)

When Sprout retries a failed action, the stream makes the loop visible so the parent watches recovery in real time instead of seeing a generic "trying again" message. The failure reason appears briefly before the retry step slides in.

Spec

.typing-state with domain tags KN, CN, AC. Transitions: typing to message (crossfade), avatar repositions from row to column layout.

  • CSS file: chat-messages.css
  • Typing to message: .sender-indicator-typing and .typing-cta fade out, then .typing-state is removed so the group reverts to column layout. Avatar moves from group-level into .msg-row.
  • Live Action Stream: Sprout-only. The .sender-indicator-typing subline cycles through action steps with slide transitions (translateY -6px out, +6px in). Steps cycle every ~900ms. Co-parent always shows static "Typing...".
  • Error recovery: When Sprout retries a failed action, the stream shows the failure reason and retry attempt (e.g. "Retry: checking schedule..." after "Failed: calendar unavailable").
