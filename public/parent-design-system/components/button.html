Overview

General-purpose action button used across connector cards, chat actions, and inline CTAs. The secondary hierarchy (default) uses a bordered white style, primary uses the brand color for emphasis, and destructive uses red for dangerous actions.

Anatomy

  • .btn - Base button (secondary hierarchy, sm size)
  • .btn-sm - Small size (6px vertical padding, text-sm)
  • .btn-md - Medium size (8px vertical padding, text-md)
  • .btn-primary - Brand solid background, white text
  • .btn-destructive - Red solid background, white text
  • :disabled - Gray background, muted text (all hierarchies)
  • .loading - Fixed width, shows .btn-spinner inside
  • .done - Tinted background, colored border and text matching hierarchy
  • .btn-spinner - 20px animated spinner

Usage

Use inside connector rows, cards, and chat messages. Secondary is for default actions (Connect), primary for emphasized CTAs (Save, Authorize), and destructive for danger actions (Disconnect, Delete). The done state confirms a completed action.

Pattern: Button stack

Use .btn-stack to group buttons vertically with 8px gap. A stack supports up to 3 buttons. Buttons auto-size to md and stretch to fill the content column. Inside .msg-group.msg-incoming the stack auto-indents past the avatar. When one button completes, add .done to it and .faded to the siblings.

3-button stack (inline permission request)

The 3-button stack is used for inline permission requests before Sprout takes a sensitive action. Sprout shows a preview of what will happen (e.g. "Sprout wants to send an email. Allow?") and the parent can approve, deny, or modify. Stack them in hierarchy order: primary (approve) on top, destructive (deny) in the middle, secondary (modify) at the bottom. Limit to 3 buttons maximum.

Default
Done (approved)
