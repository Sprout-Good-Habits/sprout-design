Scope Sheet

Permission scope toggles for a connected API service. Shows what Sprout can and can't access, with a master "All tools" toggle and individual permission rows.

Overview

The scope sheet is a bottom sheet that extends the existing artifact-sheet pattern. It displays permission toggles for an API connection, allowing parents to control what Sprout can access. It has two modes: permissions (normal use) and authorizing (empty sheet during OAuth redirect).

Anatomy

  • .scope-sheet - Modifier on .artifact-sheet (93% height, custom padding)
  • .scope-sheet-back - Pill-shaped back button (44px, border-tertiary, shadow)
  • .scope-sheet-close - Pill-shaped X close button (hidden by default, visible in authorizing state)
  • .scope-row - Permission row with text + toggle
  • .scope-row-text - Container for name and description
  • .scope-row-name - Permission name (text-md, medium weight)
  • .scope-row-desc - Permission description (text-sm, text-secondary)
  • .scope-row-master - Modifier for the "All tools" master row
  • .scope-toggle - Toggle switch container (44x24)
  • .scope-slider - Toggle track and knob
  • .scope-sheet.authorizing - Empty state during OAuth redirect

Usage

Used as a bottom sheet inside the device screen. Extends .artifact-sheet with custom header (back/close buttons) and body (permission rows with toggles). The authorizing state shows an empty sheet with only the title and a close button.
