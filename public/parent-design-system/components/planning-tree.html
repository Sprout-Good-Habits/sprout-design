Planning Tree

Multi-step plan with checkbox-style progress tracking. A persistent checklist that updates in real time as Sprout executes each step.

Overview

Persistent plan that lives next to the conversation. Sprout proposes the steps, marks them done as it goes, and revises the tree if the plan changes mid-execution. Closer to a working document than a transient progress indicator.

Anatomy

  • .planning-tree - Card container (white, rounded-20px, subtle shadow)
  • .planning-tree-header - Header row with icon and title
  • .planning-tree-header-icon - 24px clipboard SVG icon
  • .planning-tree-title - Header title text (medium weight, quaternary color)
  • .planning-tree-body - Step list container
  • .planning-item - Individual step row
  • .planning-item-icon - 24px status icon container
  • .planning-item-content - Text column (title + optional description)
  • .planning-item-title - Step title text
  • .planning-item-desc - Optional step description text

Step states

  • .pending - Queued, not yet started (outlined circle, disabled color)
  • .active - Currently executing (spinning blue arc)
  • .done - Completed (green circle with check)
  • .revised - Plan was edited mid-flight (yellow circle with dot)
  • .fail - Step abandoned (red circle with x)

Live demo

Usage

Place inside a .msg-group.msg-incoming as a sibling of the .msg-row. The card auto-indents 40px to align with the message bubble (past the avatar). Build with createPlanningTree(title, steps) in JS.
