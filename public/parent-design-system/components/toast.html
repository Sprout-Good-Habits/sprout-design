Overview

Toasts are floating notifications that appear above the chat content. They fade in, display briefly, then auto-dismiss with a fade-out and upward slide. Use them for confirmations ("Email sent"), errors ("Failed to send"), clipboard actions ("Copied"), and time-sensitive alerts ("Session expires in 2h"). They are non-blocking and require no user interaction to dismiss.

Anatomy

  • .banner - Base container with fade transition, floating card surface (white bg, soft shadow, 16px radius)
  • .banner.show - Visible state (opacity 1, pointer-events auto)
  • .banner.hiding - Exit animation (opacity 0, translateY -20px)
  • .banner-icon - 20px leading icon (success, error, info)
  • .banner-text - Message text (14px semibold, text-secondary)
  • .banner-undo - Optional action link (brand-500, no border)
  • .banner-reconnect - Variant with spinner for connection states
  • .reconnect-spinner - 20px animated spinner (brand border)
  • .banner-toast - Generic toast positioning
  • .banner-delete - Delete confirmation with undo action

Usage

Place inside .device-screen with absolute positioning. Toggle .show to display. Add .hiding before removing .show for the exit animation. Auto-dismiss after 3-5 seconds depending on content.

Spec: Toast

.banner · AC, KN, CN

Variants: success (check icon), error (x icon, red), info (info icon, yellow), copy (copy icon), delete (trash icon + undo link), reconnecting (spinner).

Behavior: Fade in on .show, auto-dismiss after 3-5s, fade out + slide up on .hiding. Non-blocking, no user action required to dismiss.

Surface: White bg, 16px radius, soft shadow (0 1px 8px rgba(0,0,0,.06)). Positioned absolutely inside device screen.
