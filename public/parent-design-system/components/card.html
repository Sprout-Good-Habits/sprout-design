Overview

The Card is the universal row component for representing a single entity inline in chat or in management views. Connectors, saved skills, contacts, milestones all use this one pattern.

Every card shares the same anatomy. The only thing that changes is what goes in each slot: what icon, what text, what trailing element.

Anatomy

  • .card / .card-row - Container. Use .card standalone, .card-row inside a .card-list.
  • .card-icon - 40x40 leading icon. Image, emoji on tinted bg (.card-icon-tinted), SVG, or avatar.
  • .card-body - Flex column with .card-label (14px medium) and optional .card-desc (12px regular).
  • .card-badge - Optional trailing pill. Variants: .badge-info, .badge-success, .badge-warning, .badge-neutral.
  • .btn - Optional trailing action button (from button.css). Connect, Configure, Reconnect, etc.
  • .card-overflow - Optional vertical dots for overflow menu (Rename, Delete, Disconnect, etc.).

Trailing elements are mutually exclusive in simple cases, but Button + Dots can appear together (e.g. Configure + overflow on a connected service).

Icon variants

  • Image: Service logo (.card-icon with <img>). Use .card-icon-inset for logos that need padding (e.g. Gmail).
  • Emoji on tinted bg: .card-icon.card-icon-tinted. An emoji character centered on a colored background. Good for skills, milestones.
  • SVG: Inline SVG inside .card-icon. For generic or system icons.

Usage in chat

Cards are inserted as siblings to the bubble inside .msg-col, just like the old connector card pattern. They sit below the text and can be wider than the bubble (330px vs 264px).
