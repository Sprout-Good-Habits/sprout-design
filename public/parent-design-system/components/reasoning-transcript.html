Reasoning Transcript

Collapsible block showing Sprout's internal reasoning. Collapsed by default with duration, click to expand and read the full reasoning prose. Opened via "Show thinking" on a Sprout message.

Overview

The reasoning transcript surfaces Sprout's internal monologue so parents can understand why Sprout said what it said. It appears as a collapsible block: collapsed by default showing duration and token count, expanded to reveal the full reasoning prose with highlights and a final decision line.

States: streaming, collapsed, expanded, with-highlights.

CSS modifiers: .streaming, .expanded.

Content variants: knowledge-lookup, connection-debug, action-planning, safety-check.

Anatomy

  • .reasoning-transcript - Outer container (white card, 12px radius, 1px border)
  • .rt-header - Clickable row: icon + label + meta + chevron
  • .rt-icon - 18px circle, colored by content variant
  • .rt-label - "Thought for Ns" or variant-specific label
  • .rt-meta - Token count or additional metadata
  • .rt-chevron - Expand/collapse indicator, rotates 180deg when open
  • .rt-body - Hidden by default, revealed on expand
  • .rt-prose - Italic reasoning text
  • .rt-highlight - Amber-highlighted key terms within prose
  • .rt-decision - Bold final decision line, separated by border-top

Sheet variant

When triggered from the "Show thinking" button on a Sprout message, the reasoning transcript opens as a bottom sheet with a timeline of discrete steps. Place .reasoning-transcript-scrim and .reasoning-transcript-sheet inside .device-screen. Toggle .open on both to show.

  • .reasoning-transcript-scrim - Semi-transparent overlay behind the sheet
  • .reasoning-transcript-sheet - Slide-up sheet container
  • .reasoning-transcript-header - Header with label and total time
  • .reasoning-transcript-body - Scrollable body area for steps
  • .reasoning-step - Individual step container
  • .reasoning-step-timeline - Visual container for dot and line
  • .reasoning-step-dot - Circle dot on the timeline
  • .reasoning-step-line - Vertical connecting line between steps
  • .reasoning-step-content - Title and description text area
  • .reasoning-step-time - Elapsed time for the step

Content variants

The reasoning transcript adapts its icon color and prose content based on what Sprout is reasoning about. Each variant uses the same anatomy but signals a different type of internal reasoning.

Knowledge Lookup

Sprout is retrieving and synthesizing information from its knowledge base to answer a parent's question.

💡
Thought for 4s 128 tokens ▲
The parent is asking about screen time guidelines for a 6-year-old. AAP recommends consistent limits and co-viewing. Emma's current setting is 45 min/day which falls within the recommended range. I should affirm their current approach and offer age-specific tips. Decision: Affirm current limits, suggest co-viewing activities

Connection Debug

Thought for 7s 203 tokens ▲
The Google Calendar connector returned a 403 Forbidden on the events.list scope. The OAuth token was issued 2024-12-01 and the parent has not re-authenticated since the scope change on 2025-03-15. The token is valid but missing the calendar.readonly scope. I need to prompt re-authorization rather than showing a generic error. Decision: Prompt parent to reconnect Google Calendar with updated permissions
