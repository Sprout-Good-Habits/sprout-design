Gap: intent-colored action buttons

The catalog models action buttons by intent (.amber=consent, .violet=connect, .red=destructive). The parent DS models them by state (.action-btn-primary + .loading / .done / .faded). These two models don't overlap.

Proposal: add .action-btn-amber, .action-btn-violet, and .action-btn-red as intent modifiers in chat-action-buttons.css, keeping -primary / -secondary as the defaults. Demo below uses this extension (scoped to this page) so you can see what it would look like before committing.

Decisions made during this port

  1. Avatar — kept the 🌱 TossFace emoji as a fallback here, but real usage should drop in the parent DS .avatar Rive canvas (canvas.rive-sprout) or silhouette SVG. Catalog's inline emoji avatars aren't grounded or branded.
  2. Typing indicator — dropped the bouncing dots; parent DS is a fading text label. This is a visual change the catalog should absorb, not the other way around.
  3. Action button intents — added .action-btn-amber/-violet/-red as a proposed extension to chat-action-buttons.css. If rejected, the catalog needs to stop encoding intent in color and pick a different signal (icon, copy, or confirm sheet).
  4. Attachments inside Sprout messages — replaced catalog's .thinking-attach with parent DS .tool-usage + .tool-usage-details. Same idea, already a real component.
  5. Sender label — catalog omits sender labels entirely. Parent DS keeps them (for co-parent disambiguation and first-message-of-a-group). Ported accordingly.
