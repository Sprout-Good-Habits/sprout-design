Type

Five typographic roles cover everything in parent chat. Pick the role first (body, label, detail, caption, sheet title), then the role tells you the size.

The 13px tier is the elephant

The "Detail" role is real: confirm-card rows, tool-usage labels, and feature item lists all reach for 13px. It sits between caption (12) and label (14), and there is no token for it. Two ways forward.

Option A
Legitimize 13px
Add --font-size-text-detail at 13px to the scale. Components that already use 13px stay where they are. Pro: matches reality. Con: adds a step to a scale that otherwise doubles cleanly.
Option B
Migrate to 14px
Treat 13px as drift. Push these slots up to --font-size-text-sm. Pro: keeps the scale clean. Con: row density tightens; needs a visual review.

This page does not pick. New work should default to 14px (label) or 12px (caption); reach for 13px only when matching an existing component.

What we don't use in chat

The full token scale goes from 12px up to display-2xl at 72px. Parent chat caps at 16px body. Anything larger only shows up in the design system docs themselves (page titles, hero copy).

  • No display-* sizes in chat. If a chat surface needs to feel like a "headline", it should still cap at 16px bold and use the 24px --radius-4xl sheet to set the stage.
  • No text-xl (20) or text-lg (18). There is currently no slot for these in chat. If a future feature needs one, treat that as a deliberate decision worth flagging.

Special cases

Status bar (17px)

The iOS status bar uses 17px semibold for the time. This is platform-grade, not a content type. Leave it hardcoded inside status-bar.css and don't reach for it elsewhere.

Artifact version badge (10px)

The version pill on artifact-card is 10px. This is a one-off badge, not a typographic role. If a second use case appears, it's a candidate for a shared "badge" pattern rather than a new font-size token.

Always pair with line-height and weight

Every size on this page has a matching line-height token. Never mix. The pairings are documented in full at kid-design-system / typography. The chat-relevant ones:

  • --font-size-text-md (16) goes with --line-height-text-md (24)
  • --font-size-text-sm (14) goes with --line-height-text-sm (20)
  • --font-size-text-xs (12) goes with --line-height-text-xs (16)
  • Sender labels and chip titles use --font-weight-semibold (600). Sheet titles and confirm-card titles use --font-weight-bold (700). Body and input use --font-weight-regular (400).

Drift to flag

Open questions
  • 13px appears in 4+ places with no shared token. See "the elephant" above.
  • 15px bold on confirm-card-title sits oddly between 14 and 16. Likely should round to 16 (text-md bold) to match the sheet titles.
  • Tool usage details and reasoning-step time use 12px hardcoded rather than the --font-size-text-xs token. Same value, missing reference.
  • Hardcoded line-heights (e.g. 18 / 20px) appear in tool-usage and reasoning-transcript. Consume the --line-height-text-* tokens instead.
